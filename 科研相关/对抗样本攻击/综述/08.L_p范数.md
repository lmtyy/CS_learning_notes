用最通俗的话说，L_p 范数就是“衡量两个向量（或两张图）差了多少”的一种尺子。p 是这把尺子的“刻度方式”。在对抗攻击里，我们常用它来限制“扰动有多大”。

核心直觉
- 你有原始输入 x 和被改动后的输入 x'，它们的差是 δ = x' − x。
- L_p 范数就是用不同的方式给 δ 一个“大小”：
  - L1：把每个位置改了多少的绝对值加起来（总改动量）。
  - L2：像勾股定理那样合成一个长度（均衡看整体改动）。
  - L∞：看改动最大的那个位置有多大（最大单点改动）。
  - L0（常用但严格说不算范数）：数一数有多少个位置被改动了（改动的个数/稀疏度）。

简单数字例子
- 假设 δ = [0.2, −0.1, 0]（三个位置的改动）
  - L1 = |0.2| + |−0.1| + |0| = 0.3
  - L2 = sqrt(0.2^2 + 0.1^2 + 0^2) ≈ 0.224
  - L∞ = max(|0.2|, |−0.1|, |0|) = 0.2
  - L0 = 有两个位置非零 → 2

几何直觉（在二维里）
- 以原点为中心、半径为 ε 的“Lp 球”的形状：
  - L1：菱形
  - L2：圆形
  - L∞：正方形
- 这表示“允许的扰动范围”。例如 L∞ ≤ ε 意味着每个坐标都不能超过 ε。

在对抗攻击里的意思
- 约束 “||x' − x||_p ≤ ε” 就是“扰动不要超过 ε”。
- 常见选择与含义：
  - L∞：每个像素（或特征）最多改一点点。图像很常用（如 ε=8/255）。
  - L2：整体改动能量小，但单点可能稍大一些。
  - L0：只允许改动很少的点（追求稀疏扰动），比如只改少数像素或少数特征位。
- 为什么常用 L∞ 在图像上？因为“每个像素都只改很小”通常肉眼不容易察觉，容易控制感知变化。

图像 vs 安全场景
- 图像：输入就是像素，L∞/L2 很好用，和“看起来差不多”较接近。
- 安全（离散/可执行）数据：L_p 不一定贴合真实约束。更常用：
  - L0（改动位数少）或操作级约束（只能插入不影响功能的操作）。
  - 例如：只能把 0/1 特征里的少数 0 变 1；或只能“追加字节/插入无害 API 调用”。

实用小抄
- 常用 ε（数据在[0,1]时）：
  - MNIST：L∞ ε≈0.1–0.3
  - CIFAR-10：L∞ ε=8/255（≈0.031），也会用 2/255、4/255
- 选哪个 p？
  - 想控制“最大单点改动” → L∞
  - 想控制“整体能量/长度” → L2
  - 想让“改动尽量少的点” → L0（或限制改动点数K）

两个易错点
- 刻度别弄错：你的数据是[0,1]还是[0,255]？同样的 ε 在两种刻度下意义差很多。
- L_p 不等于“人眼感知”：它只是方便的数学尺子，不一定完全匹配人类视觉或功能约束。在安全任务里，常需要额外的“可执行/协议合法”检查。
