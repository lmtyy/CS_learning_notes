### 一、什么是页表？

**页表是一个映射表，它的核心功能是记录每一个“虚页”被放在哪个“实页”框里。**

可以把这想象成一个**酒店前台管理系统**：

*   **虚页** = 预订了房间的客人。
*   **实页** = 酒店里实际的、有号码的房间。
*   **页表** = 前台的**入住登记表**，记录了“客人A -> 房间101”、“客人B -> 房间203”这样的对应关系。

课件中的定义非常精准：
> 存储管理软件（操作系统）根据主存情况为每个运行程序建立一个记录该程序的虚页调入主存时被安排在主存哪个实页位置，即虚页号与实页号的对照表。

**关键点：**
*   **每个程序都有自己的页表**。因为每个程序都有自己的虚拟地址空间。
*   **页表存储在内存的固定区域**。由操作系统的内存管理模块负责维护。
*   **页表项按虚页号排列**。这就像一个数组，下标就是虚页号，可以直接索引。

---

### 二、结合课件例子理解

课件给出了一个非常清晰的例子：**程序A有4个虚页（0, 1, 2, 3）**。

我们来看它的页表：

| 虚页号 | 实页号 |
| :--- | :--- |
| 0 | 1 |
| 1 | 3 |
| 2 | 8 |
| 3 | 4 |

**这个页表告诉我们什么？**

它告诉我们程序A的虚拟页在物理内存中的实际位置：
*   程序A的第 **0** 号虚页，被放在了物理内存的第 **1** 号页框中。
*   程序A的第 **1** 号虚页，被放在了物理内存的第 **3** 号页框中。
*   程序A的第 **2** 号虚页，被放在了物理内存的第 **8** 号页框中。
*   程序A的第 **3** 号虚页，被放在了物理内存的第 **4** 号页框中。

**地址转换过程（以访问程序A的虚地址 `[虚页号=2][页内地址=D]` 为例）：**

1.  CPU发出虚地址，MMU将其拆分为 `虚页号=2` 和 `页内地址=D`。
2.  MMU找到程序A的页表，去查虚页号2对应的项。
3.  从页表中查到：`虚页号2 -> 实页号8`。
4.  MMU将虚页号 `2` 替换为实页号 `8`，而**页内地址 `D` 保持不变**。
5.  于是，新的物理地址就是 `[实页号=8][页内地址=D]`。
6.  CPU用这个物理地址去访问内存条，就能拿到正确的数据。

### 三、页表项的完整结构

课件中提到了一个非常重要的概念，在表格上方写着：
`表修替换入改控制其位位位 他 实页号`

这看起来有些乱码，但它想表达的是，一个页表项（Page Table Entry, PTE）里并**不只有实页号**，还包含了很多重要的控制位。一个完整的页表项通常包括：

| 组成部分 | 作用 |
| :--- | :--- |
| **实页号（物理页框号）** | **核心部分**，指明了虚拟页对应的物理页框号。 |
| **有效位/存在位** | **最重要的位**。为1表示该虚页**已调入主存**（即上表中的映射有效）；为0表示该虚页**还在硬盘上**，访问它会引发**缺页异常**。 |
| **修改位** | 为1表示该页**曾被写入过**，是“脏”的。当此页需要被替换时，必须把它写回硬盘。如果为0（干净的），则可以直接丢弃。 |
| **访问位** | 记录该页是否被访问过（读或写）。用于**页面替换算法**（如LRU算法）参考。 |
| **保护位** | 控制访问权限，如：只读、可读可写、可执行等。 |

**为什么需要这些控制位？**
*   **有效位**：是触发缺页中断的开关。
*   **修改位**：决定了页面被换出时是否需要写回硬盘，影响效率。
*   **访问位**：帮助操作系统智能地决定当内存不足时，应该替换掉哪个“不常用”的页面。

### 总结

这一页课件让你看到了虚拟存储器魔法的幕后核心：

1.  **页表是地址转换的字典**，它存储了 `虚页号 -> 实页号` 的映射。
2.  **每个程序都有独立的页表**，由操作系统为其建立和维护。
3.  **页表项内容丰富**，除了核心的实页号，还有用于管理、保护和替换策略的各种控制位。
