**全局生效**指的是：一旦在任何一个 JSX 文件中 import 了一个 CSS 文件，这个 CSS 文件中定义的所有样式规则就会应用到**整个网页的所有组件和元素**，而不仅仅局限于引入它的那个组件。

## 具体表现：

### 1. 影响所有组件
```jsx
// 在 App.js 中引入
import './global.css';

// global.css 内容：
.button {
  background: blue;
  color: white;
}
```

这个 `.button` 样式会影响到：
- `App` 组件中的按钮
- `Header` 组件中的按钮  
- `Footer` 组件中的按钮
- **应用中所有其他组件**中的按钮

### 2. 影响所有 HTML 元素
```css
/* 在任何一个 CSS 文件中 */
p {
  color: red;
  font-size: 16px;
}
```
这个规则会让**整个网页中所有的 `<p>` 标签**都变成红色、16px 字体。

### 3. 样式冲突示例
```jsx
// ComponentA.js
import './ComponentA.css';
// ComponentA.css 内容：
.container { margin: 10px; }

// ComponentB.js  
import './ComponentB.css';
// ComponentB.css 内容：
.container { margin: 20px; }
```

结果：两个 `.container` 样式冲突，后加载的会覆盖先加载的，导致样式不一致。

## 为什么是全局的？

这是因为在 Webpack 等打包工具中，import 的 CSS 文件最终会被合并到：
- 一个或多个全局的 `<style>` 标签中，或者
- 一个全局的 CSS 文件中

然后通过 `<link>` 标签应用到整个 HTML 页面。

## 解决方案：

使用 **CSS Modules** 来限制作用域：
```jsx
import styles from './App.module.css';
// 这里的样式 ONLY 在这个组件中生效
```

**总结**：全局生效 = 样式规则对整个网页的所有元素都有效，不受组件边界限制。
