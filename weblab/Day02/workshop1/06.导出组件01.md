好的，这是 JavaScript 的模块系统概念。我来给你解释一下什么是"导出"和"导入"。

## 什么是导出（Export）？

导出就是**从一个文件中暴露一些东西（变量、函数、组件）**，让其他文件可以引入和使用。

## 两种导出方式

### 1. 默认导出（Default Export）
一个文件**只能有一个**默认导出。

```jsx
// Header.jsx
function Header() {
  return <header>网站头部</header>;
}

export default Header; // 默认导出
```

**引入时：**
```jsx
import Header from './Header'; // 可以任意命名
import MyHeader from './Header'; // 这样也可以
```

### 2. 命名导出（Named Export）
一个文件可以有**多个**命名导出。

```jsx
// UI.jsx
export function Button() { // 命名导出
  return <button>按钮</button>;
}

export function Card() { // 另一个命名导出
  return <div className="card">卡片</div>;
}

const constantValue = 100;
export { constantValue }; // 也可以先定义后导出
```

**引入时：**
```jsx
import { Button, Card } from './UI'; // 必须用原名称，用花括号
import { Button as MyButton } from './UI'; // 可以用 as 重命名
```

## 混合导出示例

```jsx
// Layout.jsx
// 默认导出
export default function Layout({ children }) {
  return <div className="layout">{children}</div>;
}

// 命名导出
export function Sidebar() {
  return <aside>侧边栏</aside>;
}

export function Content() {
  return <main>主要内容</main>;
}
```

**引入时：**
```jsx
import Layout, { Sidebar, Content } from './Layout';
// Layout 是默认导出，Sidebar 和 Content 是命名导出
```

## 实际使用示例

```jsx
// 文件1: User.jsx
export default function User() { // 默认导出
  return <div>用户组件</div>;
}

export function UserList() { // 命名导出
  return <div>用户列表</div>;
}

export const userRoles = ['admin', 'user']; // 命名导出
```

```jsx
// 文件2: App.jsx
import User, { UserList, userRoles } from './User';
// User 是默认导出，UserList 和 userRoles 是命名导出

function App() {
  return (
    <div>
      <User />
      <UserList />
      <p>角色: {userRoles.join(', ')}</p>
    </div>
  );
}
```

## 记忆技巧

- **默认导出**：`export default xxx` → `import 任意名字 from '路径'`
- **命名导出**：`export { xxx, yyy }` → `import { xxx, yyy } from '路径'`
- **一个文件只能有一个默认导出，但可以有多个命名导出**

这样设计让代码组织更清晰，组件复用更方便！
