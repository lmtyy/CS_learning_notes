问得非常好！这些都是初学者常见的困惑。让我逐一澄清：

## 1. 一个文件只有一个组件吗？

**不一定！** 有两种情况：

### 情况A：一个文件一个组件（推荐）
```jsx
// Header.jsx - 只包含 Header 组件
function Header() {
  return <header>网站头部</header>;
}
export default Header;
```

### 情况B：一个文件多个组件
```jsx
// UI.jsx - 包含多个相关组件
// 主要组件（默认导出）
export default function Button() {
  return <button>主要按钮</button>;
}

// 辅助组件（命名导出）
export function SmallButton() {
  return <button className="small">小按钮</button>;
}

export function IconButton() {
  return <button className="icon">图标按钮</button>;
}
```

## 2. 组件要和文件一个名字吗？

**推荐但不强制**，为了代码清晰：

```jsx
// ✅ 推荐：文件名和组件名一致
// Header.jsx
function Header() { ... }
export default Header;

// ❌ 不推荐但可以工作：文件名和组件名不同
// Header.jsx
function MyHeader() { ... }
export default MyHeader;
```

## 3. 默认导出的名字要和组件名一模一样吗？

**不需要！** 这是最容易混淆的点：

```jsx
// Button.jsx
function SuperButton() { // 组件名是 SuperButton
  return <button>按钮</button>;
}

export default SuperButton; // 这里写组件名，但...

// 在其他文件引入时：
import Button from './Button'; // 这里可以任意命名！
import MyBtn from './Button';  // 这样也可以！
import Whatever from './Button'; // 甚至这样！
```

**关键点**：默认导出时，引入的文件可以**任意命名**！

## 4. 命名导出是在导出什么？

命名导出通常用于：

### A. 导出多个相关组件
```jsx
// FormComponents.jsx
export default function Input() { // 主要组件
  return <input />;
}

export function Label() { // 辅助组件
  return <label>标签</label>;
}

export function TextArea() { // 辅助组件
  return <textarea></textarea>;
}
```

### B. 导出常量、配置、工具函数
```jsx
// constants.jsx
export const API_URL = 'https://api.example.com';
export const THEME_COLORS = { primary: 'blue', secondary: 'gray' };

// 使用：
import { API_URL, THEME_COLORS } from './constants';
```

### C. 导出多个工具组件
```jsx
// Icons.jsx
export function HomeIcon() { return <svg>...</svg>; }
export function UserIcon() { return <svg>...</svg>; }
export function SettingsIcon() { return <svg>...</svg>; }

// 使用：
import { HomeIcon, UserIcon } from './Icons';
```

## 实际项目中的常见模式

```jsx
// 模式1：一个文件一个主要组件
// UserProfile.jsx
export default function UserProfile() { ... }

// 模式2：一个主要组件 + 相关子组件  
// Form.jsx
export default function Form() { ... }
export function FormField() { ... }
export function SubmitButton() { ... }

// 模式3：纯工具集合（无默认导出）
// helpers.jsx
export function formatDate(date) { ... }
export function validateEmail(email) { ... }
```

**==简单总结==**：
- 默认导出：文件的主要东西，引入时可以任意命名
- 命名导出：文件的次要东西，引入时必须用原名称
- 文件名和组件名一致是为了代码清晰，不是强制要求
