前端初始化不仅仅是运行一行命令下载模板，更重要的是**搭建一个干净、好用、适合后续扩展的开发环境**。

对于你现在的技术栈（React + Vite），由于你需要配合 Express 后端，初始化的重点在于：**目录结构清理**、**基础库安装** 以及 **Vite 配置（尤其是代理）**。

下面是详细的“手把手”初始化指南：

---

### 第一步：创建项目脚手架

在你的项目根目录下（假设你有一个总文件夹叫 `my-project`）：

```bash
# 1. 创建名为 client 的前端项目（使用 React 模板）
npm create vite@latest client -- --template react

# 2. 进入目录
cd client

# 3. 安装依赖 (这一步会下载 node_modules)
npm install
```

此时的目录结构是 Vite 给你的默认模版，里面包含了一个计数器的 Demo。

---

### 第二步：清理“样板代码” (Clean Up)

官方模版里有很多我们不需要的 Logo、CSS 和演示代码。我们需要把它变成一张“白纸”。

1.  **删除文件**：
    *   删除 `src/assets` 文件夹下的 `react.svg`。
    *   删除 `public` 文件夹下的 `vite.svg`（可选，也可以换成你的 favicon）。

2.  **重置样式**：
    *   打开 `src/index.css`：清空内容，或者只保留 `* { margin: 0; padding: 0; box-sizing: border-box; }` 这种最基础的 Reset CSS。
    *   打开 `src/App.css`：直接清空或删除。

3.  **重写 App.jsx**：
    *   把 `App.jsx` 里的计数器代码全删了，只留最外壳。
    
    ```jsx
    // src/App.jsx 清理后长这样
    function App() {
      return (
        <div>
          <h1>项目初始化完成</h1>
        </div>
      )
    }
    
    export default App
    ```

---

### 第三步：安装“全栈开发”必备库

既然你会 React 和 Express，你肯定需要路由跳转和发送网络请求。

```bash
# 一次性安装常用的库
npm install axios react-router-dom classnames
```

*   **axios**: 比原生的 `fetch` 更好用，自动处理 JSON 数据，方便配置拦截器。
*   **react-router-dom**: React 官方路由，用于写多页面应用（单页应用中的多页面切换）。
*   **classnames** (可选): 写 CSS 类名时非常有用的一个小工具，可以用条件判断来动态添加 class。
*   *(进阶选装)*: 如果不想手写 CSS，可以安装 `tailwindcss` 或组件库 `antd` / `mui`。初学者建议先手写 CSS 或使用 **Less/Sass** (`npm install -D sass`)。

---

### 第四步：配置 Vite (关键步骤)

这一步是前后端交互的核心。打开根目录下的 `vite.config.js`。

我们需要配置两个东西：
1.  **路径别名 (@)**：避免出现 `../../../components/Button` 这种深层引用。
2.  **反向代理 (Proxy)**：**非常重要**。解决开发阶段浏览器报的“跨域(CORS)”错误。

```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path' // 需引入 path 模块

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      // 设置 @ 指向 src 目录
      '@': path.resolve(__dirname, './src'),
    },
  },
  server: {
    // 代理配置
    proxy: {
      '/api': {
        target: 'http://localhost:3000', // 后端 Express 的地址
        changeOrigin: true,
        // 如果后端接口没有 /api 前缀需要去掉，可以在这里 rewrite
        // rewrite: (path) => path.replace(/^\/api/, '') 
      },
    },
  },
})
```

**配置好 Proxy 后的效果：**
前端代码里写 `axios.get('/api/users')`，浏览器虽然是发给 `localhost:5173`（前端），但 Vite 会偷偷把它转发给 `localhost:3000`（后端）。**这完美解决了开发时的跨域问题。**

---

### 第五步：规划目录结构

不要把所有文件都堆在 `src` 下，按照功能拆分文件夹。一个标准的 React 目录结构如下：

```
src/
├── assets/          # 静态资源 (图片, 字体)
├── components/      # 公共组件 (Button, NavBar)
├── pages/           # 页面组件 (Login, Home, Dashboard) - 配合路由使用
├── router/          # 路由配置
├── services/        # 统一管理 API 请求 (axios 封装)
├── hooks/           # 自定义 Hooks
├── utils/           # 工具函数 (格式化时间等)
├── App.jsx
└── main.jsx
```

你可以现在就手动创建这几个文件夹（`components`, `pages`, `services`）。

---

### 第六步：封装 Axios (建议)

在 `src/services` 下新建 `request.js`。别每次都在组件里写 `axios.get`，封装一下更好管理。

```javascript
// src/services/request.js
import axios from 'axios';

// 创建实例
const instance = axios.create({
  baseURL: '/api', // 配合 vite.config.js 的代理
  timeout: 5000,
});

// 请求拦截器 (比如发送请求前带上 token)
instance.interceptors.request.use(config => {
  // const token = localStorage.getItem('token');
  // if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// 响应拦截器 (比如统一处理错误)
instance.interceptors.response.use(
  response => response.data,
  error => {
    console.error('请求出错了:', error);
    return Promise.reject(error);
  }
);

export default instance;
```
