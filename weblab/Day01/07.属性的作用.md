这个问题问得非常好！答案是：**两者都有，但大多数属性不会直接显示在网页的可见内容中**。

让我详细为你解释：

## 1. 不会显示在网页中的属性（大多数）

### 功能性和编程性属性
```html
<!-- 这些属性不会在页面上显示任何内容 -->
<div id="header" class="container" data-user-id="123">
    这是可见内容
</div>

<a href="about.html" target="_blank" title="关于我们页面">关于我们</a>

<img src="photo.jpg" alt="描述文字" width="300" height="200">
```

**用户看到的是：**
- "这是可见内容"
- "关于我们"（链接文字）
- 图片本身

**不会看到：**
- `id="header"`
- `class="container"` 
- `data-user-id="123"`
- `target="_blank"`
- `title="关于我们页面"`（除非鼠标悬停）
- `alt="描述文字"`（除非图片加载失败）

## 2. 会间接影响显示的属性

### 通过CSS或JavaScript影响显示
```html
<!-- class 和 style 属性会影响外观但不直接显示 -->
<p class="highlight" style="color: red;">这段文字是红色的</p>

<!-- 用户看到：红色的"这段文字是红色的" -->
<!-- 但看不到 class="highlight" 和 style="color: red;" -->
```

### 表单元素的 value 属性
```html
<input type="text" value="默认值">
<!-- 用户会看到输入框中有"默认值"文字 -->

<textarea name="message">默认内容</textarea>
<!-- 用户会看到文本域中有"默认内容" -->
```

## 3. 会在特定情况下显示的属性

### title 属性（悬停显示）
```html
<a href="#" title="这是一个提示信息">悬停看我</a>
```
**正常情况：** 用户看到"悬停看我"
**鼠标悬停时：** 会显示"这是一个提示信息"的提示框

### alt 属性（图片加载失败时显示）
```html
<img src="missing.jpg" alt="图片描述">
```
**图片正常加载：** 用户看到图片
**图片加载失败：** 用户看到"图片描述"文字

### placeholder 属性（输入框提示文字）
```html
<input type="text" placeholder="请输入用户名">
```
**用户看到输入框中有灰色的"请输入用户名"**
**当用户开始输入时，提示文字消失**

## 实际演示

### 示例代码：
```html
<!DOCTYPE html>
<html>
<head>
    <title>属性显示演示</title>
    <style>
        .special { background-color: yellow; }
    </style>
</head>
<body>
    <!-- 这些属性值不会直接显示 -->
    <div id="main" class="container" data-info="secret">
        <h1 title="这是一个标题">欢迎来到我的网站</h1>
        
        <!-- alt 只在图片加载失败时显示 -->
        <img src="image.jpg" alt="网站标志图片" width="200">
        
        <!-- placeholder 会显示但非实际值 -->
        <input type="text" placeholder="在这里输入内容" value="实际值">
        
        <!-- value 会显示 -->
        <input type="text" value="这是默认文本">
        
        <!-- class 通过CSS间接影响显示 -->
        <p class="special">这个段落有特殊样式</p>
        
        <!-- 链接的文本内容显示，但属性不显示 -->
        <a href="page.html" target="_blank" title="点击访问">访问页面</a>
    </div>
</body>
</html>
```

### 用户实际看到的页面：
```
欢迎来到我的网站
[图片显示]
[输入框显示"在这里输入内容"]
[输入框显示"这是默认文本"]
这个段落有特殊样式（黄色背景）
访问页面
```

**用户看不到：**
- `id="main"`
- `class="container"`
- `data-info="secret"`
- `title="这是一个标题"`
- `alt="网站标志图片"`
- `href="page.html"`
- `target="_blank"`

## 属性在开发者工具中的可见性

虽然用户在前端看不到这些属性，但开发者可以在浏览器开发者工具中查看：

1. 按 F12 打开开发者工具
2. 在 Elements 面板中可以看到所有属性

```
<div id="main" class="container" data-info="secret">
    <h1 title="这是一个标题">欢迎来到我的网站</h1>
    <img src="image.jpg" alt="网站标志图片" width="200">
</div>
```

## 总结

| 属性类型 | 是否直接显示 | 作用 |
|---------|-------------|------|
| `id`, `class` | ❌ 不显示 | 用于CSS和JavaScript |
| `href`, `src` | ❌ 不显示 | 定义资源路径 |
| `alt` | ⚠️ 条件显示 | 图片失败时显示 |
| `title` | ⚠️ 悬停显示 | 鼠标悬停时提示 |
| `value` | ✅ 显示 | 表单元素的默认值 |
| `placeholder` | ✅ 显示 | 输入框提示文字 |
| 标签内容 | ✅ 显示 | 用户直接看到的内容 |

**核心要点：**
- **属性主要是编程作用** - 为浏览器、CSS、JavaScript提供信息
- **内容才是给用户看的** - 标签之间的文字内容
- **有些属性在特定情况下会显示** - 如 `title`, `alt`, `placeholder`

所以你说的"编程中的作用"是完全正确的！属性主要是为了让网页能够正常工作、样式美观、交互丰富，而不是直接展示给用户看的内容。
