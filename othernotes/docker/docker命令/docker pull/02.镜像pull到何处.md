很好的问题！`docker pull` 拉取的镜像存储在 Docker 引擎管理的**本地存储区域**中，这个位置取决于你的操作系统和存储驱动。

---

### 一、镜像存储的默认位置

不同操作系统的默认存储路径：

#### **Linux** (最常见)
```bash
/var/lib/docker/
```
在这个目录下，镜像主要存放在：
- `/var/lib/docker/image/` - 镜像元数据和层信息
- `/var/lib/docker/overlay2/` - 实际的镜像层文件（如果使用 overlay2 存储驱动）

#### **macOS** 和 **Windows** (Docker Desktop)
由于 macOS 和 Windows 通过虚拟机运行 Docker，镜像存储在一个 Linux VM 中，但你可以通过 Docker Desktop 界面管理存储位置。

- **macOS**: `~/Library/Containers/com.docker.docker/Data/vms/0/data/`
- **Windows**: `C:\ProgramData\Docker\`

---

### 二、如何查看和管理镜像存储

#### 1. 查看已拉取的镜像
```bash
docker images
```
这会列出所有本地存储的镜像，显示它们的大小和存储情况。

#### 2. 查看 Docker 磁盘使用情况
```bash
docker system df
```
输出示例：
```
TYPE            TOTAL     ACTIVE    SIZE      RECLAIMABLE
Images          5         3         1.2GB     450MB (37%)
Containers      3         1         25MB      25MB (100%)
Local Volumes   2         1         150MB     150MB (100%)
Build Cache     0         0         0B        0B
```

#### 3. 查看详细磁盘使用情况
```bash
docker system df -v
```

### 三、镜像存储的实际结构

Docker 使用**分层存储**机制，理解这一点很重要：

#### 示例：当你拉取一个镜像时
```bash
docker pull nginx:alpine
```

这个镜像可能由 5 个层组成：
1. **基础层** (Alpine Linux)
2. **系统工具层**
3. **Nginx 二进制文件层**
4. **配置文件层**
5. **启动脚本层**

这些层都独立存储在 `/var/lib/docker/overlay2/` 下的不同目录中。

**关键优势**：如果另一个镜像也使用相同的基础层（比如另一个基于 Alpine 的镜像），Docker 不会重复下载和存储这个层，而是**共享使用**。

---

### 四、管理镜像存储

#### 1. 删除不再使用的镜像
```bash
# 删除单个镜像
docker rmi nginx:alpine

# 删除所有未被使用的镜像
docker image prune

# 强制删除所有未被使用的镜像（包括被容器使用的）
docker image prune -a
```

#### 2. 清理整个 Docker 系统
```bash
# 删除所有停止的容器、未使用的网络、构建缓存和悬空镜像
docker system prune

# 彻底清理（包括未使用的镜像和数据卷）
docker system prune -a --volumes
```

---

### 五、修改默认存储位置

如果默认位置磁盘空间不足，可以修改存储路径：

#### **Linux** 修改方法：
1. 停止 Docker 服务：
   ```bash
   sudo systemctl stop docker
   ```
2. 修改 `/etc/docker/daemon.json`：
   ```json
   {
     "data-root": "/new/path/to/docker"
   }
   ```
3. 重启 Docker：
   ```bash
   sudo systemctl start docker
   ```

#### **Docker Desktop** (macOS/Windows)：
在 Docker Desktop 设置中直接修改磁盘镜像位置。

---

### 总结

- **拉取到哪里**：`/var/lib/docker/` (Linux) 或 Docker Desktop 管理的虚拟磁盘中
- **存储方式**：分层存储，相同的基础层会被共享
- **如何查看**：使用 `docker images` 和 `docker system df`
- **如何管理**：定期使用 `docker system prune` 清理不需要的镜像

简单来说，**Docker 帮你管理了一个本地的"镜像仓库"，你不需要关心具体文件位置，只需要通过 `docker images` 命令来查看和管理即可。**
