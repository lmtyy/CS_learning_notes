好的！我们专注讲挂载卷的**两种核心方式**，这是最常用也最重要的。

---

## 方式一：挂载主机目录（Bind Mount）

### 语法
```bash
docker run -v /主机/绝对/路径:/容器内路径 镜像名
```

### 特点
- **直接使用主机上的真实目录**
- 主机和容器**共享同一个物理位置**
- 适合：开发环境、配置文件、代码同步

### 示例
```bash
# 把主机的 /home/user/app 目录挂载到容器的 /app 目录
docker run -v /home/user/app:/app nginx

# 使用当前目录（开发时最常用）
docker run -v $(pwd)/html:/usr/share/nginx/html nginx
```

### 验证
```bash
# 在主机上操作
echo "Hello from Host" > /home/user/app/test.txt

# 在容器内查看
docker exec 容器名 cat /app/test.txt
# 输出：Hello from Host
```

---

## 方式二：挂载数据卷（Volume）

### 语法
```bash
docker run -v 卷名:/容器内路径 镜像名
```

### 特点
- **由 Docker 管理的存储空间**
- 位置在 `/var/lib/docker/volumes/卷名`
- 适合：数据库文件、生产环境、需要备份的数据

### 示例
```bash
# 创建并挂载名为 mysql_data 的数据卷
docker run -v mysql_data:/var/lib/mysql mysql

# Docker 会自动创建不存在的卷
docker run -v my_app_data:/app/data nginx
```

### 验证
```bash
# 查看所有数据卷
docker volume ls

# 查看数据卷详情
docker volume inspect mysql_data
```

---

## 两种方式的对比总结

| 特点 | 主机目录挂载 | 数据卷挂载 |
|------|-------------|------------|
| **存储位置** | 主机指定路径 | Docker 管理 (`/var/lib/docker/volumes/`) |
| **创建方式** | 需确保路径存在 | Docker 自动创建 |
| **适用场景** | 开发环境、代码同步 | 生产环境、数据库 |
| **备份迁移** | 手动管理 | Docker 命令管理 |
| **性能** | 直接访问主机文件系统 | 经过 Docker 存储驱动 |

---

## 如何选择？

### 选择 **主机目录挂载** 当：
- 你在开发，需要实时修改代码
- 需要编辑配置文件
- 想要直接访问和修改容器内的文件

### 选择 **数据卷挂载** 当：
- 运行数据库（MySQL、PostgreSQL等）
- 生产环境部署
- 需要 Docker 来管理备份和迁移
- 不想暴露主机目录结构

---

## 最简单的记忆方法

```bash
# 开发用这个：-v /主机路径:/容器路径
docker run -v $(pwd)/code:/app nginx

# 生产用这个：-v 卷名:/容器路径  
docker run -v db_data:/var/lib/mysql mysql
```

就这两种方式，掌握了就解决了 90% 的数据持久化需求！
