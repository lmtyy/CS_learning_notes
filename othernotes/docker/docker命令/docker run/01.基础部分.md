好的！我们先从 `docker run` 命令最简单、最核心的用法开始讲起。这是 Docker 中最重要也最常用的命令。

---

### 一、命令概述

`docker run` 的核心功能是：**从指定的镜像创建并启动一个新的容器**。

简单来说，它做了两件事：
1.  **创建**：基于镜像创建一个新的容器实例
2.  **启动**：启动这个容器，运行其中指定的应用程序

---

### 二、最基本语法

```bash
docker run [选项] 镜像名称 [命令]
```

---

### 三、最常用的简单场景

#### 场景1：直接运行一个镜像（前台运行）

```bash
# 运行 Ubuntu 容器，并在容器内执行 `echo "Hello World"`
docker run ubuntu echo "Hello World"
```

**发生了什么？**
- Docker 从本地查找 `ubuntu` 镜像，如果没有就自动从 Docker Hub 下载
- 创建并启动一个 Ubuntu 容器
- 在容器内执行 `echo "Hello World"`
- 命令执行完毕后，容器自动停止

#### 场景2：让容器在后台运行

```bash
# 在后台运行 Nginx 服务器
docker run -d nginx
```

**参数说明：**
- **`-d`** 或 **`--detach`**：让容器在**后台运行**，不占用当前终端

#### 场景3：为容器起个名字

```bash
# 运行一个名为 "my-web" 的 Nginx 容器
docker run -d --name my-web nginx
```

**参数说明：**
- **`--name`**：为容器指定一个易记的名称（否则 Docker 会随机生成一个名字）

#### 场景4：端口映射（最常用！）

```bash
# 将主机的 8080 端口映射到容器的 80 端口
docker run -d -p 8080:80 --name my-nginx nginx
```

**参数说明：**
- **`-p`** 或 **`--publish`**：`主机端口:容器端口`
- **作用**：让你可以通过主机的 8080 端口访问容器内的 80 端口服务
- **验证**：在浏览器访问 `http://localhost:8080` 就能看到 Nginx 欢迎页面

---

### 四、最简单的完整示例

让我们一步步完成一个最简单的 Web 服务部署：

```bash
# 1. 下载 Nginx 镜像（如果还没有的话）
docker pull nginx

# 2. 运行容器：后台运行 + 命名 + 端口映射
docker run -d --name my-website -p 8080:80 nginx
```

就这么两行命令，你就拥有了一个正在运行的 Nginx 服务器！

---

### 五、查看运行结果

```bash
# 查看正在运行的容器
docker ps

# 查看容器日志
docker logs my-website

# 停止容器
docker stop my-website

# 再次启动容器（注意：这里是 start，不是 run）
docker start my-website
```

---

### 六、总结：最简版的 `docker run`

记住这个**万能公式**，就能应对大部分简单场景：

```bash
docker run -d --name <给你的容器起个名> -p <主机端口>:<容器端口> <镜像名>
```

**实际例子：**
```bash
# 运行一个 WordPress 网站
docker run -d --name my-blog -p 8080:80 wordpress

# 运行一个 Redis 缓存
docker run -d --name my-cache -p 6379:6379 redis
```

**核心要点：**
- `docker run` = 创建 + 启动
- `-d` = 后台运行
- `--name` = 给容器起名
- `-p` = 端口映射（让外部能访问）

先掌握这4个核心参数，你就已经能用 Docker 做很多有用的事情了！更复杂的选项（如数据卷、环境变量、网络等）我们可以在后续深入讲解。
