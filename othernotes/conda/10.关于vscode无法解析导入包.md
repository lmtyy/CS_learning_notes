根据你的描述，VSCode未使用你安装包的Conda环境（Python 3.9），而是使用了另一个Python 3.13的解释器，这确实是导致`pydub`导入失败的**最常见原因**。

你可以按照以下步骤来检查和修正这个问题。

### 🔧 检查并修正VSCode的Python解释器

核心操作是确保VSCode使用的解释器与你安装`pydub`的Conda环境路径完全一致。

1.  **确认Conda环境路径**
    在你的命令行（如终端、Anaconda Prompt）中，激活你的Conda环境，并运行以下命令来获取该环境中Python解释器的**绝对路径**。
    ```bash
    conda activate your_env_name  # 激活你的conda环境
    python -c "import sys; print(sys.executable)"
    ```
    记下输出的路径，例如：`C:\Users\YourName\anaconda3\envs\your_env_name\python.exe`。

2.  **在VSCode中选择正确解释器**
    *   在VSCode中打开你的项目文件夹。
    *   按下 **`Ctrl+Shift+P`** (Windows/Linux) 或 **`Cmd+Shift+P`** (Mac) 打开命令面板。
    *   输入并选择 **`Python: Select Interpreter`**。
    *   在弹出的列表中，寻找与你第一步中记下的路径匹配的解释器（通常会显示为 `Python 3.9.x ('your_env_name': conda)`）并选中它。

### 🛠️ 如果问题依旧存在

如果在切换解释器后，VSCode底部的状态栏确实显示为正确的环境，但导入错误仍然存在，可以尝试以下进阶操作：

*   **重启VSCode和语言服务器**：关闭VSCode然后重新打开。如果还有错误，可以按下 `Ctrl+Shift+P`，输入 `Python: Restart Language Server` 并执行。
*   **手动配置项目设置**：如果解释器切换不彻底，可以在项目根目录的 `.vscode/settings.json` 文件中（如没有则新建），手动指定解释器路径：
    ```json
    {
        "python.defaultInterpreterPath": "C:\\Users\\YourName\\anaconda3\\envs\\your_env_name\\python.exe"
    }
    ```
    将路径替换为你第一步获取的实际路径。
*   **检查终端是否激活环境**：确保VSCode内置的终端也使用了正确的环境。在终端中，你应该能看到类似 `(your_env_name)` 的前缀。如果没有，可以手动运行 `conda activate your_env_name`。
