WSL2 与 Windows 文件系统交互主要通过以下几种方式：

## 1. **访问 Windows 文件**
### 从 WSL2 访问 Windows 文件
```
# 访问 Windows C 盘
cd /mnt/c/

# 访问用户文件夹
cd /mnt/c/Users/你的用户名/

# 访问其他分区
cd /mnt/d/   # D 盘
```

### 从 Windows 访问 WSL2 文件
```
# 在资源管理器中输入
\\wsl$

# 或使用路径
\\wsl$\Ubuntu   # 如果是 Ubuntu 发行版
\\wsl$\Debian   # 如果是 Debian 发行版
```

## 2. **文件路径转换**
### WSL 路径转 Windows 路径
```bash
# 使用 wslpath 工具
wslpath -w /home/username/documents
# 输出：C:\Users\你的用户名\AppData\Local\Packages\...

# Windows 路径转 WSL 路径
wslpath -u "C:\Users\username\file.txt"
```

## 3. **共享环境变量**
```bash
# 在 WSL2 中访问 Windows 环境变量
echo $PATH
# 包含 Windows PATH

# 查看特定 Windows 环境变量
export WINDOWS_USER=$(cmd.exe /c "echo %USERNAME%" 2>/dev/null | tr -d '\r')
```

## 4. **互操作命令**
### 在 WSL2 中运行 Windows 命令
```bash
# 直接运行 .exe 文件
notepad.exe
explorer.exe .

# 运行 PowerShell 命令
powershell.exe Get-Date

# 执行 cmd 命令
cmd.exe /c dir
```

### 在 Windows 中运行 WSL 命令
```cmd
# 在 CMD/PowerShell 中
wsl ls -la
wsl python3 script.py
```

## 5. **配置文件交互**
### 常用配置文件路径映射
```bash
# WSL2 主目录
/home/用户名

# Windows 用户目录
/mnt/c/Users/用户名/

# 项目共享建议：放在 /mnt/c/ 下便于双向访问
```

## 6. **性能优化建议**
```bash
# 1. 将需要高性能操作的文件放在 WSL2 文件系统中
#    Linux 文件系统（/home/）性能更好

# 2. 将需要双向访问的文件放在 Windows 分区
#    /mnt/c/ 性能稍差但便于共享

# 3. 避免在 /mnt/ 下编译大型项目
```

## 7. **常见工具集成**
```bash
# VS Code 集成
code .  # 自动调用 Windows 版 VS Code

# Git 配置
# 可以使用 Windows 的 Git，也可以安装 Linux 版 Git
```

## 8. **网络交互**
```bash
# 访问 Windows 本地服务
curl http://localhost:3000  # 直接访问 Windows 运行的服务器

# Windows 访问 WSL2 服务
# 需要配置端口转发或使用 WSL2 IP 地址
```

## 注意事项
1. **文件权限**：Windows 文件在 WSL 中可能显示为 777
2. **行尾符**：注意 CRLF（Windows）和 LF（Unix）的区别
3. **性能**：跨文件系统操作可能较慢
4. **符号链接**：可能需要额外配置

## 快捷方式
```bash
# 在 WSL 中快速打开 Windows 资源管理器
explorer.exe .
# 或
wslview .
```

这些方法可以让你在 WSL2 和 Windows 之间无缝地进行文件操作和工具调用。
