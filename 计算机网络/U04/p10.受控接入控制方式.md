这页PPT的核心思想是：**实现“轮流”，既可以通过一个“老大”来指挥，也可以让大家遵守一套平等的规则。**

---

### 两大控制范式

我们可以将这两种控制方式对比理解：

| 特性 | 集中式控制 | 分散式控制 |
| :--- | :--- | :--- |
| **核心思想** | 有一个**中央控制器**来管理和调度所有节点。 | 所有节点**地位平等**，通过预定的分布式规则自行协调。 |
| **比喻** | **“交通警察”** <br> 警察站在路口，指挥每辆车何时可以通行。 | **“传话筒”** <br> 一个会议室里只有一个话筒，大家按顺序传递，拿到话筒的人才能发言。 |
| **典型协议** | **轮询** | **令牌传递** |
| **工作方式** | 主节点依次询问从节点：“你要发送数据吗？” | 一个特殊的“令牌”帧在网络中循环，拿到令牌的节点才能发送数据。 |
| **优点** | - **控制力强**，主节点可以优先处理重要节点。<br>- 实现相对简单。 | - **无单点故障**，健壮性高。<br>- **公平**，所有节点机会均等。 |
| **缺点** | - **单点故障**：主节点失效则全网瘫痪。<br>- **性能瓶颈**：主节点可能成为网络瓶颈。 | - **令牌管理复杂**：需要处理令牌丢失、重复等问题。<br>- 节点加入/离开网络时逻辑更复杂。 |

---

### 为什么这个分类很重要？

这个分类揭示了网络设计的根本权衡：

1.  **简单性与健壮性的权衡**
    *   **集中式 (轮询)**：实现逻辑相对简单，但系统脆弱。
    *   **分散式 (令牌传递)**：实现逻辑复杂，但系统更健壮。

2.  **效率与公平的权衡**
    *   **集中式 (轮询)**：主节点可以进行智能调度（如优先处理数据量大的节点），可能提升整体效率，但可能不公平。
    *   **分散式 (令牌传递)**：严格按照固定顺序轮流，非常公平，但可能不够灵活。

### 知识框架回顾

现在，你可以把第四章的这个部分清晰地放进知识框架里：

*   **问题：** 多路访问信道 → 冲突
*   **解决方案之一：** 受控接入 → 核心思想是“轮流”
*   **“轮流”的两种实现路径：**
    *   **集中式控制** → **轮询** (像老师点名)
    *   **分散式控制** → **令牌传递** (像传话筒)
