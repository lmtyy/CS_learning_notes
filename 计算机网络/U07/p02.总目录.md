## 第 7 章：传输层核心概览

### 7.1 传输层服务 (The Transport Service)

传输层位于网络层之上，应用层之下。它的核心任务是：**为运行在不同主机上的应用进程提供逻辑通信。**

* **网络层 vs 传输层**：网络层负责“主机到主机”的通信（像邮局把信送到你家楼下）；传输层负责“进程到进程”的通信（像室友把信亲手交到你手里）。
* **复用与分用**：多个应用同时联网时，传输层能准确识别数据属于哪个程序。

### 7.2 端口号 (Port Number)

如果说 IP 地址标识的是互联网上的“房子”，那么**端口号**就是进入房子后的“房号”。

* **范围**： 到 。
* **常见端口**：HTTP (80), HTTPS (443), FTP (21)。
* **作用**：让传输层知道要把收到的数据交给哪个特定的 Socket。

### 7.3 TCP (Transmission Control Protocol)

**关键词：可靠、面向连接、慢工出细活。**
这是计网考试的重灾区，你需要重点掌握：

* **三次握手**：建立连接，确保双方都有发送和接收的能力。
* **四次挥手**：优雅地释放连接，确保数据传输完毕。
* **特性**：
1. **可靠性**：通过确认号（ACK）和重传机制保证数据不丢、不乱。
2. **流量控制**：通过“滑动窗口”防止发送太快把接收方淹没。
3. **拥塞控制**：检测网络状况，防止整个网络堵塞。



### 7.4 UDP (User Datagram Protocol)

**关键词：不可靠、无连接、快准狠。**

* **特点**：没有握手过程，数据发出去就不管了，头部开销极小。
* **应用场景**：实时视频通话、在线游戏、DNS 查询。这些场景宁愿掉一两帧画面，也不希望因为等待重传而导致卡顿。

| 特性 | TCP | UDP |
| --- | --- | --- |
| **连接性** | 面向连接 | 无连接 |
| **可靠性** | 100% 可靠 | 尽力而为（不可靠） |
| **速度** | 较慢（开销大） | 极快（开销小） |

### 7.5 套接字编程 (Socket Programming)

这是将理论付诸实践的部分。套接字（Socket）是应用层与传输层之间的**抽象接口（API）**。

* **流程**：你需要学习如何使用 Python、C 或 Java 调用 `socket()`, `bind()`, `listen()`, `accept()`, `connect()` 等函数。
* **理解**：编写代码时，你会创建一个 Socket，指定它是 TCP 类型（SOCK_STREAM）还是 UDP 类型（SOCK_DGRAM）。
