这三页课件展示了 **TCP 段（Segment）** 的报文首部格式，它是整个传输层最复杂也最重要的结构。

你可以把这 20 个字节的首部想象成一张“快递单”，上面不仅写了地址（端口），还写了包裹的顺序、大小和加急信息。

---

### 1. TCP 首部的基本结构 (image_1b8e0a)

这是 TCP 段的核心组成部分。在没有选项的情况下，首部长为 **20 字节**。

* **源端口（Source Port）** 和 **目的端口（Destination Port）**：占据了最开始的 4 个字节，每个 16 位。
* **序列号（Sequence Number）**：32 位，标记当前段发送的是字节流里的哪些字节。
* **确认号（Acknowledgment Number）**：32 位，告诉对方“我已经收到了哪些字节，我期待你的下一个字节从哪开始”。

### 2. 源端口 (image_1b8e28)

* **定义**：标识发送端主机上发起通信的具体应用程序。
* **作用**：当对方（服务器）回包时，它会把这个端口作为“目的端口”，确保回复能准确送到你电脑里的那个特定程序（如浏览器的一个 Tab 页）。

### 3. 目的端口 (image_1b90d1)

* **定义**：标识接收端主机上接收数据的具体应用程序。
* **作用**：数据到达服务器后，操作系统会拆开 TCP 头部，根据这个端口号（例如 80）把数据交给对应的 Web 服务器程序。

---

### 🎓 考试重点：5 元组 (5-tuple)

这三页课件结合前面的内容，其实是在告诉你一个网络连接是如何被**唯一确定**的。
在计算机内部，每个 TCP 连接都对应一个 **5 元组**：

1. **源 IP 地址**（在 IP 头里）
2. **源端口号**（在 TCP 头里）
3. **目的 IP 地址**（在 IP 头里）
4. **目的端口号**（在 TCP 头里）
5. **协议类型**（TCP）

这就是为什么你的电脑可以同时开 10 个网页访问同一个服务器，而数据不会串门——因为每个网页连接的“源端口”都是系统随机分配的，互不相同。
