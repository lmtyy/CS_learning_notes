这三张新的课件正式引入了网络编程中的灵魂概念——**套接字（Socket）**。它将我们之前讨论的 IP 地址和端口号有机地结合在了一起。

以下是详细解读：

---

### 1. IP 地址与端口号的分工 (7.2.2 Socket)

这张课件清晰地定义了两者在寻找“最终目的地”时扮演的不同角色：

* **目的 IP 地址**：负责在全世界范围内的众多主机中，准确地找到那台**特定的主机**。
* **端口号**：在选定主机后，负责定位该主机上运行的**某一个具体进程**。
* **总结**：IP 帮你找到“哪栋楼”，端口帮你找到“哪一户”。

### 2. 什么是套接字 (Socket)？

这张课件给出了 Socket 的物理/逻辑结构：

* **定义**：Socket 被视为通信的**端点（End Point）**。
* **组成公式**：**Socket = IP 地址 + 端口号**。
* 例如：图中 IP `131.6.23.13` 加上端口 `1500`，就构成了一个唯一的 Socket `(131.6.23.13, 1500)`。


* **通信前提**：两个进程如果要进行通信，必须在各自的端点都创建一个 Socket。

### 3. Socket 通信实例

这张图展示了客户端与服务器交互时的真实 Socket 样貌：

* **客户端 Socket (Client Socket)**：
* 由客户端 IP（如 `128.6.4.194`）和系统随机分配的**临时端口号**（如 `1234`）组成。


* **服务器 Socket (Server Socket)**：
* 由服务器 IP（如 `18.22.67.7`）和提供服务的**熟知端口号**（如 Web 服务的 `80`）组成。


* **通信过程**：数据就在这两个 Socket 确定的“管道”之间流动。

---

### 🎓 关键知识点总结

作为计网学习者，你需要记住这个核心逻辑：

1. **传输层**使用端口号来区分进程。
2. 将**网络层**提供的 IP 地址与**传输层**提供的端口号“绑定”在一起，就得到了 **Socket**。
3. **Socket 是应用层与传输层之间的编程接口（API）**。当你写代码调用 `connect()` 或 `send()` 时，你本质上是在操作这个 Socket。
