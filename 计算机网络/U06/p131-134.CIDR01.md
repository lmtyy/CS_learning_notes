这四张课件讲述了一个非常完整的逻辑故事，主题是 **“如何打破传统的分类限制，把两个小的C类网络合并成一个大网络”**。

这在网络技术中被称为 **CIDR (无类别域间路由)** 或 **Supernetting (超网)**。

我按顺序为你把这个故事串起来：

### 第一幕：进退两难的困境 (Slide 1)
**对应课件：**

* **场景：** 一个中型公司有 **500台** 主机。
* **尴尬：** 按照旧的分类规则（Classful）：
    * **选 C 类？** 一个 C 类只能容纳 254 台。**太小了 (Shortage)，不够用。**
    * **选 B 类？** 一个 B 类能容纳 65534 台。**太大了 (Wasting)，极度浪费。**
* **思考：** 我们能不能申请 **2个 C 类** 地址（254 x 2 = 508），刚好够用？

---

### 第二幕：简单的拼凑是行不通的 (Slide 2)
**对应课件：**

* **尝试：** 我们真的拿到了两个连续的 C 类地址块：
    * `202.118.0.0`
    * `202.118.1.0`
* **操作：** 我们把这 500 台电脑接在同一个办公室里（同一个物理 LAN），一半配成 0.x 的IP，一半配成 1.x 的IP。
* **问题：** 使用默认的 C 类掩码 **`255.255.255.0`**。
* **结果：** **无法通信！**
    * 电脑 A (`202.118.0.1`) 也就是 0 号房间的人。
    * 电脑 B (`202.118.1.1`) 也就是 1 号房间的人。
    * 因为掩码是 `/24`，它们会检查 IP 的前三段。前三段不一样（0 vs 1），电脑就会认为：“你不是我这个局域网的人”，于是试图去找路由器转发，而不会直接通信。
* **结论：** 物理上在一起，逻辑上却被掩码隔离了。

---

### 第三幕：神奇的解决方案——超网 (Slide 3)
**对应课件：**

* **核心技巧：** 修改子网掩码 (Subnet Mask)。
* **操作：** 我们不使用默认的 `255.255.255.0`，而是改成 **`255.255.254.0`**。
* **原理分析：**
    * `255` (二进制 11111111)
    * `254` (二进制 1111111**0**) $\leftarrow$ **关键就在这最后一位变0了！**
    * 原来的 `/24` 变成了 **`/23`**。这意味着网络号的判断标准“向左缩了一位”。
* **合并效果：**
    * `202.118.0.0` 的第三段是 `0000000 0`
    * `202.118.1.0` 的第三段是 `0000000 1`
    * 因为掩码最后一位是 **0**（表示这一位不看网络号，看主机号），路由器现在只看前 7 位。而前 7 位这两个地址是**一模一样**的（都是 0000000）。
* **结果：** 两个原本独立的 C 类网络，在逻辑上被**合并**成了一个大网络。所有 500 台主机都认为彼此是“邻居”，**可以互相通信了！**

---

### 第四幕：理论总结——CIDR 的诞生 (Slide 4)
**对应课件：**

* **背景：** 这张图解释了为什么要搞这么麻烦。
* **Classful scheme (有类方案) 的罪状：**
    1.  **分配不均：** 就像买鞋子只有“童鞋(C类)”和“巨人鞋(B类)”，大部分普通人(中型企业)没鞋穿。
    2.  **B类枯竭：** 因为中型企业都被迫申请 B 类，导致 B 类地址快发光了。
    3.  **C类闲置：** 大量的 C 类没人要，因为太小。
* **CIDR (无类别域间路由)：** 就是我们在第三幕里做的事。它打破了 A、B、C 类的边界，允许我们要多少拼多少（比如拼两个 C 类），从而解决了 IP 地址短缺的危机。

### 总结
这四张图讲述了网络工程中**“变通”**的智慧：当标准的规则（C类太小，B类太大）不适用时，我们通过**缩短子网掩码**（Supernetting），把多个小网络合并成一个大网络来满足需求。
