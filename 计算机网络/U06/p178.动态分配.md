这张图紧接着刚才的“静态分配”，讲解了我们现在最常用的**“动态分配 (Dynamic Assignment)”**。

如果不使用管理员手动去一台台填 IP，我们怎么上网？答案就是靠**协议**自动获取。这张 PPT 列出了动态分配技术的**“进化史”**，从老祖宗到现代标准：

### 1. RARP (Reverse Address Resolution Protocol) —— 老祖宗
* **名字含义**：**逆向**地址解析协议。
* **它的作用**：还记得 ARP 是“知道 IP 找 MAC”吗？RARP 正好反过来，它是**“知道自己的 MAC，问我的 IP 是多少”**。
* **场景**：这通常用于早期的无盘工作站（Diskless Workstation）。电脑一开机，不知道自己是谁，只知道网卡上的 MAC 地址，于是大喊一声：“我是 MAC xx-xx...，谁能给我分配个 IP？”
* **缺点**：功能太单一，只能给个 IP，给不了子网掩码、网关这些信息，所以现在基本淘汰了。

### 2. BOOTP (Bootstrap Protocol) —— 中兴之主
* **名字含义**：引导程序协议。
* **它的进步**：它比 RARP 聪明，不仅能告诉你 IP，还能顺便告诉你**默认网关**在哪里、DNS 服务器是谁。
* **场景**：用于计算机通过网络启动（而不是通过硬盘启动）的场景。
* **局限**：虽然能配全套信息，但它的分配通常是“静态映射”的，还是不够灵活。

### 3. DHCP (Dynamic Host Configuration Protocol) —— 现代王者
* **名字含义**：动态主机配置协议。
* **地位**：这是目前互联网的**绝对标准**，你现在连 WiFi 能直接上网，全靠它。
* **核心特点**：
    * **即插即用**：不需要任何手动配置。
    * **租约机制 (Lease)**：DHCP 分配的 IP 不是永久给你的，而是“租”给你的。你下线了，这个 IP 就会被收回，分给别人用。这就解决了 IP 地址不够用的问题。
    * **全自动**：IP、掩码、网关、DNS 一股脑全发给你。

### 💡 总结
这张 PPT 其实展示了一个**进化路线**：
**RARP (只问 IP)** ➡️ **BOOTP (能问更多配置)** ➡️ **DHCP (还能动态租用与回收)**。

考试的时候，只要记住 **DHCP** 是现在的使用的协议，而 **RARP** 是 ARP 的逆过程即可。
