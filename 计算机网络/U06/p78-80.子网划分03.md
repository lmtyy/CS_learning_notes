这三页是 **子网掩码计算的实例和默认掩码介绍**。我来为你详细解析：

## **一、示例计算：172.16.65.100 / 255.255.240.0**

### **1. 题目解析**
```
给定：IP地址 172.16.65.100，子网掩码 255.255.240.0
求：该IP所在的子网地址
```

### **2. 逐步计算（第一页）**
**步骤1：转换为二进制**
```
IP地址：  172  .  16  .  65  .  100
二进制：10101100.00010000.01000001.01100100
```

**步骤2：子网掩码二进制**
```
掩码：    255  .  255  .  240  .  0
二进制：11111111.11111111.11110000.00000000
注意：20个1（/20），12个0
```

**步骤3：AND运算**
```
IP:     10101100.00010000.01000001.01100100
掩码:   11111111.11111111.11110000.00000000
AND结果:10101100.00010000.01000000.00000000
```

**步骤4：转换回十进制**
```
10101100 = 172
00010000 = 16  
01000000 = 64
00000000 = 0

子网地址：172.16.64.0
```

### **3. 结果解读**
- IP `172.16.65.100/20` 属于子网 `172.16.64.0/20`
- 该子网范围：`172.16.64.1` ~ `172.16.79.254`
- 广播地址：`172.16.79.255`

## **二、CIDR表示法：172.16.65.100 / 20（第三页）**

### **1. /20 的含义**
- `/20` 表示子网掩码有**20个连续的1**
- 二进制：`11111111.11111111.11110000.00000000`
- 十进制：`255.255.240.0`

### **2. 计算验证**
```
/20 = 前20位是网络+子网部分
      |← 20位 →| |← 12位 →|
掩码：11111111 11111111 1111 0000 00000000
      255      .255      .240    .0
```

第三页的计算与第一页完全相同，只是强调了**20位网络部分**和**12位主机部分**。

---

## **三、==默认子网掩码==（第二页）**

### **1. 什么是默认掩码？**
- 在 **==不划分子网==** 的情况下使用的掩码
- 由IP地址的**类别**决定
- 也称为"自然掩码"或"分类掩码"

### **2. 各类地址的默认掩码**

| 地址类别 | 默认掩码 | CIDR表示 | 网络位数 | 主机位数 |
|----------|----------|----------|----------|----------|
| **A类** | 255.0.0.0 | /8 | 8位 | 24位 |
| **B类** | 255.255.0.0 | /16 | 16位 | 16位 |
| **C类** | 255.255.255.0 | /24 | 24位 | 8位 |

### **3. 默认掩码的意义**
```
A类地址：1.0.0.1 ~ 126.255.255.254
默认掩码：255.0.0.0
含义：前8位是网络部分，后24位是主机部分

例如：10.20.30.40/8
网络地址：10.0.0.0
主机地址：10.20.30.40
```

## **四、关键概念：借位**

### **1. 从默认掩码到子网掩码**
```
B类默认：255.255.0.0 (/16)
         网络部分（16位） | 主机部分（16位）

划分子网：借4位主机位
新掩码：255.255.240.0 (/20)
         网络+子网（20位） | 主机部分（12位）
```

### **2. 借位计算**
```
原始B类：16位网络，16位主机
新掩码：/20
借位数 = 20 - 16 = 4位
子网数 = 2^4 = 16个子网
每个子网主机数 = 2^(32-20) - 2 = 2^12 - 2 = 4094台
```

## **五、实际网络规划示例**

### **场景：公司使用172.16.0.0/16**
```
需求：划分16个部门子网，每个部门约200台电脑

计算：
1. 需要16个子网 → 2^4 = 16 → 借4位
2. 新掩码：/16+4 = /20 (255.255.240.0)
3. 每个子网主机数：2^(32-20)-2 = 4094台（远大于需求）

划分：
子网1：172.16.0.0/20   (0-15)
子网2：172.16.16.0/20  (16-31)
子网3：172.16.32.0/20  (32-47)
...
子网16：172.16.240.0/20 (240-255)
```

## **六、常见错误和注意事项**

### **1. 掩码连续性**
- 掩码中的1必须连续
- `/20` 有效：`11111111.11111111.11110000.00000000`
- 无效掩码：`11111111.11111111.00001111.00000000`（1不连续）

### **2. IP地址与掩码匹配**
- A类IP配B类掩码可能出错
- 如：`10.0.0.1/16` 技术上可行，但可能不符合网络规划

### **3. 全0和全1子网**
- 传统上，全0子网（如172.16.0.0/20）和全1子网（172.16.240.0/20）不使用
- 现代设备通常支持（`ip subnet-zero`命令）

## **七、快速判断技巧**

### **1. 看最后一个非255的掩码值**
```
掩码 255.255.240.0
关键值：240
二进制：11110000
说明：借了4位（1111有4个1）
```

### **2. 块大小计算**
```
掩码值 240 → 二进制 11110000 → 块大小 = 256-240 = 16
子网间隔：16
示例：
172.16.0.0/20
172.16.16.0/20
172.16.32.0/20
...
```

### **3. IP属于哪个子网？**
```
IP：172.16.65.100/20
关键在第三段：65
块大小：16
计算：65 ÷ 16 = 4余1 → 第4个块
子网地址：172.16.64.0（4×16=64）
```
