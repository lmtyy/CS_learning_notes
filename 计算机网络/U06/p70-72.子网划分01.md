这几页开始介绍 **子网划分（Subnetting）** 的核心概念。这是解决IP地址浪费问题的关键技术。我来为你详细解释：

---

## **一、为什么需要子网划分？**

### **1. 问题的提出（第二页表格）**
假设一个组织需要**100台主机**的网络：

| 地址类别 | 申请IP数 | 该类网络允许的IP数 | **浪费的IP数** |
|----------|----------|-------------------|---------------|
| **A类** | 100 | 16,777,214 | **16,777,114**（浪费99.999%） |
| **B类** | 100 | 65,534 | **65,434**（浪费99.85%） |
| **C类** | 100 | 254 | **154**（浪费60.6%） |

**结论**：
- A类、B类严重浪费
- C类相对好点，但还是浪费超过一半
- **分类地址太不灵活！**

---

## **二、子网划分的核心思想**

### **1. 基本概念**
- 将**一个大的网络**划分成**多个小的子网络**
- 从 **==主机部分借位==** 来创建子网
- 这些子网在外部看来仍然是一个网络

### **2. 关键操作："借位"**
```
原始分类地址：
┌────────── 网络部分 ──────────┐ ┌────── 主机部分 ──────┐
110xxxxx xxxxxxxx xxxxxxxx xxxxxxxx  ← C类地址示例

子网划分后：
┌────────── 网络部分 ──────────┐ ┌─子网─┐ ┌─主机─┐
110xxxxx xxxxxxxx xxxxxxxx xxx xxxxxxx
         原网络部分             借来的位  剩余主机位
```

---

## **三、子网划分示例（C类地址）**

### **1. 原始C类地址**
```
IP地址：192.168.1.0/24
二进制：11000000.10101000.00000001.00000000
划分： 网络部分（24位） | 主机部分（8位）
可用主机：254台（192.168.1.1 ~ 192.168.1.254）
```

### **2. 划分子网（借2位）**
```
借2位主机位作为子网位：
网络部分（24位） | 子网部分（2位） | 主机部分（6位）
新结构：192.168.1.0/26（24+2=26）

结果：
- 子网数量：2^2 = 4个子网
- 每个子网主机数：2^6 - 2 = 62台主机
- 总共可用主机：4 × 62 = 248台（接近原始的254）
```

---

## **四、子网划分的好处**

### **1. 减少IP浪费**
回到100台主机的需求：
- 用C类地址：254个地址，浪费154个
- **用子网划分**：可以从B类网络中划一个合适大小的子网
  ```
  例如：从B类网络172.16.0.0/16中划分
  借9位主机位：172.16.0.0/25（或类似）
  每个子网约126台主机，基本满足需求
  ```

### **2. 提高网络管理效率**
- **广播域隔离**：减少广播流量
- **安全隔离**：不同部门在不同子网
- **故障隔离**：一个子网问题不影响其他
- **路由优化**：内部路由更精细

### **3. 保持外部路由简单**
- 对外部路由器来说，整个组织还是一个网络
- 不需要为每个子网维护独立路由表项

---

## **五、子网划分的关键要素**

### **1. ==子网掩码==（Subnet Mask）**
- 标识哪些位是网络/子网部分，哪些是主机部分
- 示例：`255.255.255.192` = `/26` = `11111111.11111111.11111111.11000000`

### **2. 子网地址计算**
```
原始网络：192.168.1.0/24
子网掩码：255.255.255.192（/26）

4个子网：
1. 192.168.1.0/26   范围：.1~.62
2. 192.168.1.64/26  范围：.65~.126
3. 192.168.1.128/26 范围：.129~.190
4. 192.168.1.192/26 范围：.193~.254
```

### **3. 每个子网的三个特殊地址**
```
以子网1（192.168.1.0/26）为例：
- 子网地址：192.168.1.0（主机全0）
- 可用地址：192.168.1.1 ~ 192.168.1.62
- 广播地址：192.168.1.63（主机全1）
```

## **六、实际应用场景**

### **场景：大学网络划分**
```
大学获得B类地址：172.16.0.0/16（65534台主机）

划分需求：
- 计算机系：500台主机
- 图书馆：300台主机  
- 行政楼：200台主机
- 教学楼：150台主机

解决方案：
借位划分，例如：
- 计算机系：172.16.0.0/23（510台）
- 图书馆：172.16.2.0/24（254台）
- 行政楼：172.16.3.0/24（254台）
- 教学楼：172.16.4.0/24（254台）
```

---

## **七、重要注意事项**

### **1. ==借位限制==**
- 主机位至少保留2位（否则无法有可用主机）
- 最大借位数 = 主机位数 - 2

### **2. 现代实践**
- 现在主要使用 **CIDR（无类别域间路由）**
- 子网划分是CIDR的前身和基础
- 概念类似，但CIDR更灵活

### **3. 规划要点**
- 预留发展空间
- 考虑网络设备（路由器接口、服务器）需要地址
- 考虑特殊地址（网络地址、广播地址）

---

## **八、一句话总结**

> **子网划分是通过"借用"主机位来创建更小、更灵活的网段，解决分类地址"不是太大就是太小"的问题，让IP地址分配更高效。**
