这页课件开启了一个新的话题：**如何获取 IP 地址 (Obtaining an Internet Address)**。

在之前的路由跳转和 ARP 实验中，我们看到的 IP 地址（比如 `172.16.1.1`）都是假设已经存在于电脑里的。那么，这些 IP 到底是怎么配置到电脑上的呢？

这页课件总结了两种主要的方法：

### 1. 静态分配 (Static Assignment)
* **含义**：由网络管理员（Admin）手动在电脑的设置里填入 IP 地址、子网掩码和网关。
* **特点**：
    * **固定不变**：一旦设好，这台机器的 IP 就永远是这个（除非你去改）。
    * **麻烦**：如果你管理 1000 台电脑，你需要一台台去配，还得小心不能配重了（IP 冲突）。
* **适用场景**：
    * **服务器、路由器、打印机**：这些设备需要被别人找到，所以地址不能变来变去。比如我们之前讲的网关路由器，它的地址通常就是固定的（如 `192.168.1.1`）。

### 2. 动态分配 (Dynamic Assignment)
* **含义**：电脑连上网络时，自动向网络中的“管家”申请一个 IP 地址。
* **背后的协议**：虽然课件这页没写，但你一定要知道，这通常是靠 **DHCP (Dynamic Host Configuration Protocol)** 协议来实现的。
* **特点**：
    * **省事**：用户什么都不用管，插上网线/连上 WiFi 就能上网。
    * **临时性**：这个 IP 通常是“租”给你的，过段时间或者重启后，可能会变。
* **适用场景**：
    * **个人电脑、手机、平板**：比如你在咖啡厅连 WiFi，没人会给你一张纸让你手动输 IP，都是自动获取的。

### 💡 知识点关联
还记得我们在 ARP 那里讲过的 `arp -a` 命令吗？那里也有 `dynamic`（动态）和 `static`（静态）的字眼。
* **ARP 的 Dynamic**：是指 **“自动学习”** 到了别人的 MAC 地址。
* **IP 的 Dynamic**：是指 **“自动获取”** 了自己的 IP 地址。

虽然层级不同，但计算机网络里用 **Dynamic** 这个词，通常都代表 **“自动化、无需人工干预”**。
