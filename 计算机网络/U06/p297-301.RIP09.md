这五页课件详细讲解了针对距离矢量路由（Distance Vector Routing）中“计数到无穷”和“路由环路”问题的**三种核心解决方案**。

这是网络工程师为了弥补 RIP 协议原生缺陷而设计的巧妙机制：

---

### 1. 路由毒化 (Route Poisoning)

当一个网络发生故障时，与其直连的路由器不会仅仅删掉该条目，而是主动“投毒”。

* **操作**：如图所示，当 `10.4.0.0` 断开时，路由器 C 立即将其距离设为 **Down**（或逻辑上的“无穷大”）。
* **通俗理解**：与其等着邻居来传谣言，不如我先大声告诉所有人：“这条路已经彻底烂掉了，千万别走！”

### 2. 定义最大值 (Defining a Maximum)

这是最简单粗暴的兜底方案。

* **设定上限**：在 RIP 协议中，将 **16** 定义为不可达（Infinity）。
* **作用**：即使发生了环路，跳数也不会无止境地增加。一旦达到 16，路由器就会彻底丢弃该包并停止更新。这防止了单个错误信息永久占用网络资源。

### 3. 水平分割 (Split Horizon)

这是防止环路最有效的原则之一。

* **核心思想**：路由器绝不会把从某个接口学到的路由信息，再通过同一个接口发回去。
* **案例分析**：如果 B 是从 C 那里听说 `10.4.0.0` 怎么走的，那么 B 在给 C 发更新时，绝对不会包含关于 `10.4.0.0` 的信息。
* **意义**：这有效切断了“互传谣言”的回路，避免了 B 和 C 之间产生数据包来回踢皮球的**路由环路 (Routing Loops)**。

### 4. 毒性反转 (Poison Reverse)

它是水平分割的一种特殊变体，通常会覆盖（Override）标准的水平分割。

* **机制**：当 B 收到 C 的毒化信息（距离为无穷大）后，B 会打破水平分割的禁令，特意向 C 回复一个“我也认为这里不可达（Infinity）”的消息。
* **目的**：确保邻居之间达成明确的一致，彻底消除任何关于该路径还可能存在的幻想。

---

### 💡 重点总结与对比

| 技术名称 | 核心逻辑 | 解决什么问题 |
| --- | --- | --- |
| **Route Poisoning** | 将故障路由设为 Infinity | 阻断谣言的产生 |
| **Split Horizon** | 不原路返回信息 | 防止双节点间的简单环路 |
| **Defining Maximum** | 跳数上限为 16 | 终止无尽的计数循环 |
| **Poison Reverse** | 显式确认不可达 | 加速全网收敛 |
