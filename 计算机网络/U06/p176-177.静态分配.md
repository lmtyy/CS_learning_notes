这次我们来讲这两张关于 **“静态分配 (Static Assignment)”** 的核心内容。

这几张图紧接着刚才的“获取 IP 地址”，详细展示了**手动配置 IP** 的具体操作和规则。

### 1. 它是长什么样的？(图 image_bfa7dd.jpg)
这张图展示了经典的 Windows（看起来像 XP 时代）网络配置界面。当管理员选择 **“静态分配”** 时，就是在这个界面操作的。

* **标志**：选中了 **"Use the following IP address" (使用下面的 IP 地址)**，而不是 "Obtain automatically" (自动获取)。
* **必填四大件**：
    1.  **IP Address (IP 地址)**：例如 `202.206.100.18`，这是这台电脑唯一的身份标识。
    2.  **Subnet mask (子网掩码)**：例如 `255.255.255.192`，它决定了局域网的大小。
    3.  **Default gateway (默认网关)**：例如 `202.206.100.1`。
        * **重点**：这就是我们之前说的**路由器**的局域网接口地址。没有它，你的电脑只能在屋里（局域网）玩，出不去互联网。
    4.  **DNS server (DNS 服务器)**：例如 `202.206.100.36`。它负责把 `www.baidu.com` 翻译成 IP 地址。

### 2. 它的规则是什么？(图 image_bfa7e3.jpg)
既然是人工分配，就不能乱填。这张图提出了一个极为重要的概念：**私有地址 (Private IP addresses)**。

* **管理员职责**：系统管理员需要手动记录和分配每一台电脑的 IP，防止冲突。
* **私有网络规范**：在家庭、学校实验室或公司内部，我们通常**不直接使用**公网 IP，而是从下面这三个**保留段**里挑地址。这些地址在互联网上是路由不通的，专门留给内部使用：
    * **A 类保留段**：`10.0.0.0` 到 `10.255.255.255`（通常用于超大型企业网络）。
    * **B 类保留段**：`172.16.0.0` 到 `172.31.255.255`（学校实验室特别爱用这个，刚才课件里的例子 `172.16.1.1` 就在这里面）。
    * **C 类保留段**：`192.168.0.0` 到 `192.168.255.255`（最常见，你家里的 WiFi 路由器一般就是 `192.168.1.x`）。
---
**💡 总结**
新发的这几张图告诉你：
如果你要手动配网（Static Assignment），你需要填 IP、掩码、网关、DNS 四大件。而且在内部组网时，你应该自觉地从 `10.x`、`172.16.x` 或 `192.168.x` 这三个私有地址池里选号码，不要去占用公网地址。
