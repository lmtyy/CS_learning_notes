这两张课件介绍了在 Windows 网络环境中，除了 DNS 之外，用于将计算机名称转换为 IP 地址的另一种机制：**WINS 服务**及其底层的 **NetBIOS 协议**。

作为计网课程的一部分，这通常是为了让你理解局域网（LAN）内部命名服务的演进。

---

### 1. NetBIOS 协议 (对应 image_29a4ec.jpg)

NetBIOS（网络基本输入/输出系统协议）是一个非常经典的局域网通信协议。

* **起源**：最初由 IBM 公司开发，主要针对只有**数十台计算机**的小型局域网设计。
* **核心功能**：在局域网中直接使用**计算机名**（例如你的电脑在“网上邻居”里显示的名字）进行通讯，就是使用的 NetBIOS 协议。
* **实际应用**：Windows 系统中早期的“网上邻居”功能就是基于此协议实现的。

### 2. WINS 服务 (对应 image_29a4f1.jpg)

WINS（Windows Internet 命名服务）可以看作是“局域网版本的 DNS”。

* **功能定位**：
* **映射关系**：它负责 **IP 地址和 NetBIOS 名**（计算机名称）之间的映射。
* **作用域**：计算机名称通常只在一个**单位内部**（如校园网或企业内网）有效。


* **与 DNS 的对比**：
* **WINS**：处理内部计算机名（如 `MyLaptop`）到 IP 的映射。
* **DNS**：处理互联网域名（如 `www.google.com`）到 IP 的映射。


* **管理工具**：在 Windows 中，你可以通过 `ipconfig` 命令来查看当前机器的 WINS 和 DNS 配置信息。

---

### 💡 核心知识点总结

| 协议/服务 | 解决的问题 | 适用范围 |
| --- | --- | --- |
| **NetBIOS** | 允许电脑通过“名字”在局域网发现彼此。 | 小型局域网。 |
| **WINS** | 实现 NetBIOS 名到 IP 的自动转换。 | 企业/单位内部网络。 |
| **DNS** | 实现全球范围内的域名解析。 | 整个互联网。 |

**学习建议：**
在现代网络中，WINS 已经逐渐被 **DNS**（通过动态更新）和 **LLMNR** 等新协议取代了。你只需要记住：**WINS 是为了在 Windows 局域网里解决“计算机名”变 IP 的老牌方案**。
