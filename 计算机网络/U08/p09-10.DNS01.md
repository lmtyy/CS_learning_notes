这两张课件正式带你进入了 **DNS (Domain Name System, 域名系统)** 的世界。作为大学生，理解 DNS 是理解互联网“导航”机制的第一步。

以下是针对这两张课件的详细解读：

---

### 1. 为什么需要 DNS？ (对应 image_29160a.jpg)

这张图用了一个非常生动的人类社会类比来解释 DNS 的必要性。

* **核心矛盾**：
* **机器的语言**：在 TCP/IP 协议中，网络设备通过 **IP 地址**（如 `202.112.0.36`）来唯一标识彼此的连接。
* **人类的习惯**：人类更擅长记忆有意义的**名称**（如 `www.edu.cn`），而不是枯燥的数字。


* **映射 (Mapping)**：
* 就像每个人都有**姓名**（便于称呼）和**身份证号**（唯一法定标识）一样。
* **DNS 的本质**：它是一个分布式数据库系统，充当互联网的“电话簿”，负责将人类可读的域名映射为机器可读的 IP 地址。



### 2. DNS 的演进：从 Host 文件到系统 (对应 image_291624.jpg)

这张图展示了域名映射的具体例子，并提到了 DNS 出现之前的原始方案。

* **映射关系示例**：
* 一个域名可以对应一个确定的 IP（如清华大学或 MIT 的例子）。
* **多对一映射**：注意图中大括号的部分，多个不同的域名（如 `www.edu.cn`, `www.net.edu.cn`）可以指向**同一个 IP 地址**。这通常是因为一台强大的服务器同时托管了多个服务。


* **Host 文件 (早期方案)**：
* **定义**：在互联网早期，所有的映射关系都存储在每台电脑本地的一个名为 `hosts` 的文本文件中。
* **局限性**：随着互联网爆炸式增长，手动更新每台电脑上的 `hosts` 文件变得不可能。
* **现代残留**：你的电脑里现在依然有这个文件（比如 Windows 的 `C:\Windows\System32\drivers\etc\hosts`），它通常用于本地测试或屏蔽特定网站。

### 💡 核心知识点总结

| 概念 | 解释 |
| --- | --- |
| **正向解析** | 将域名转换为 IP 地址（最常用的功能）。 |
| **反向解析** | 将 IP 地址转换为域名。 |
| **分布式特性** | 现在的 DNS 不是存在一台机器上，而是分布在全球成千上万台服务器中。 |
