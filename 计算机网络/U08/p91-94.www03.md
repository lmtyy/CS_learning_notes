这四张课件详细讲解了万维网（WWW）实现资源定位和内容获取的三大核心支柱：**HTML**、**HTTP** 和 **URL**。

作为计网课程的结尾部分，这些是理解现代互联网应用最直观的知识点。

---

### 1. HTML：超文本标记语言 (对应 image_52f445.jpg)

HTML 是网页的“骨架”和“外观说明书”。

* **功能**：它是为 WWW 服务器上的信息资源制作、并供浏览器呈现而约定的一种格式化语言。
* **超文本形式**：它能将文字、图片等内容组织成“超文本”。
* **用户交互**：图中展示了当鼠标悬停在带有下划线的文本（链接）上时，光标会变成“小手”形状，点击即可激活跳转。

### 2. URL：统一资源定位器 (对应 image_52f461.jpg)

URL 是互联网上每个资源的“身份证号”或“家庭住址”。

* **作用**：在 WWW 上获取信息必须指明其位置，URL 就是对信息进行定位的通用访问地址。
* **标准格式**：`应用协议类型 : // 主机名 / 路径名 / ... / 文件名`。
* **示例解析**：
* `http://www.edu.cn/`：使用 HTTP 协议访问该网站首页。
* `ftp://ftp.pku.edu.cn/pub/netscape/`：使用 FTP 协议去北京大学的服务器下载文件。



### 3. HTTP：超文本传输协议 (对应 image_52f468.jpg)

HTTP 是浏览器和服务器之间沟通的“语言”。

* **核心职能**：它是为了高效传送超文本链接所必须的一切信息而设计的协议。
* **传输层支撑**：HTTP 是**建立在 TCP 连接之上**的应用层协议，确保了网页数据传输的可靠性。
* **组成部分**：它由**资源定位**（你要什么）和**信息内容格式**（我给你什么）两部分组成。

### 4. 万维网的工作过程 (对应 image_52f710.jpg)

这张时序图完整展示了你点击一个链接到看到网页的全过程：

1. **建立连接**：浏览器首先与服务器（如 `www.tsinghua.edu.cn`）建立 **TCP 连接**。
2. **发送请求**：浏览器发送一个 **HTTP 请求报文**（问服务器要文档）。
3. **返回响应**：服务器回传一个 **HTTP 响应报文**（包含具体的 HTML 文档内容）。
4. **释放连接**：数据传完后，释放 TCP 连接。
5. **渲染呈现**：浏览器解析收到的 HTML，将其显示在屏幕上。
