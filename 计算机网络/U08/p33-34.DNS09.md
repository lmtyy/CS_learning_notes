### 1. 访问网页的完整交互流程 (对应 image_29a1a6.png)

这张图展示了一个客户（Client）访问 `www.edu.cn` 时的四个关键步骤。这实际上是**两个不同协议**协作的结果：

* **阶段一：DNS 解析（寻找地址）**
1. **请求 (Step 1)**：客户机向域名服务器发送请求，询问 `www.edu.cn` 的 IP 地址。
2. **回答 (Step 2)**：域名服务器查表后返回 IP 地址 `202.112.0.36`。


* *注：此时浏览器还什么都没看到，只是知道了目的地在哪。*


* **阶段二：HTTP 访问（获取内容）**
3.  **请求 (Step 3)**：客户机拿到 IP 后，直接向该 IP 对应的 Web 服务器发起请求，要求访问 CERNET 主页。
4.  **回答 (Step 4)**：Web 服务器将网页数据（HTML/图片等）传送给客户，你的浏览器这时才显示出网页。

---

### 2. 核心标识符的关系 (对应 image_29a1c3.jpg)

这张图通过 Windows 网络属性窗口，总结了计算机网络中三个最核心的标识符及其关系：

* **主机名 (Host Name / Domain Name)**：如 `www.edu.cn`。这是给人记的，代表逻辑身份。
* **IP 地址 (IP Address)**：图中显示的 `202.206.100.18`。这是给网络层路由用的，代表逻辑位置。
* **MAC 地址 (MAC Address)**：虽然图中未直接显示具体数值，但在数据链路层必须有它才能在局域网内传输。它代表硬件身份。

**三者的绑定关系：**

* **DNS 负责**：将“主机名”映射为“IP 地址”。
* **ARP 负责**：在局域网内将“IP 地址”映射为“MAC 地址”。
* **配置重点**：图中圈出的 **Preferred DNS server (`202.206.100.36`)** 就是你的本地域名服务器，它是所有解析请求的第一站。

---

### 💡 知识点总结（考试常考）

| 步骤 | 使用协议 | 目的 | 依赖关系 |
| --- | --- | --- | --- |
| **第一步** | **DNS** | 获取服务器 IP | 如果 DNS 挂了，输入网址将无法访问，但直接输入 IP 可能还能上。 |
| **第二步** | **HTTP/HTTPS** | 获取网页内容 | 必须先完成 DNS 解析才能进行此步。 |
