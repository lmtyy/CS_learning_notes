---

### 8.1 客户-服务器模型 (Client-Server Model)

这是应用层最经典的通信模式。

* **核心逻辑**：一方是**服务器 (Server)**，它永久运行，拥有固定 IP，等待并响应请求；另一方是**客户机 (Client)**，它主动发起请求，可以间歇性连接。
* **对比 P2P**：现在的网络也有对等模式（Peer-to-Peer），但 C/S 仍然是主流（如 Web 浏览）。

### 8.2 域名系统 (DNS - Domain Name System)

它是互联网的“电话簿”。

* **作用**：将人类易记的域名（如 `www.google.com`）转换成机器识别的 **IP 地址**。
* **结构**：采用层次树状结构（根域名服务器 -> 顶级域名服务器 -> 权限域名服务器）。
* **查询方式**：分为**递归查询**和**迭代查询**，这是考试常考的重点。

### 8.3 TELNET (远程终端协议)

这是一个比较“古老”的协议。

* **功能**：允许用户在本地电脑上登录并操作远端服务器。
* **现状**：由于它是**明文传输**，安全性极差，现在基本已被更安全的 **SSH (Secure Shell)** 取代。但在学习原理时，它是理解远程操作的入门。

### 8.4 文件传输协议 (FTP - File Transfer Protocol)

专门用于在网络上传输大文件。

* **双连接特性（重要考点）**：
1. **控制连接**（端口 21）：用于发送指令（登录、删除文件）。
2. **数据连接**（端口 20）：用于真正传输文件内容。


* 这种“带外控制”的设计让文件传输更稳定。

### 8.5 电子邮件 (Electronic Mail)

邮件系统的运作涉及多个协议：

* **SMTP**：负责把邮件“推”出去（发送）。
* **POP3 / IMAP**：负责把邮件“拉”回来（接收）。IMAP 比 POP3 更先进，支持同步状态（如在手机上已读，电脑上也显示已读）。

### 8.6 WWW (World Wide Web)

这是我们每天都在用的万维网，主要由三部分组成：

1. **HTTP 协议**：规定了浏览器和服务器怎么说话。
2. **HTML 语言**：规定了网页长什么样。
3. **URL**：规定了资源在互联网上的具体位置。
