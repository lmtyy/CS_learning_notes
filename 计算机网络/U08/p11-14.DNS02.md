以下是针对这四张课件的详细解读：

---

### 1. 层次域名空间 (Hierarchical Name Space) (对应 image_291ac5.jpg)

这张图展示了 DNS 如何组织全球亿万个域名。

* **倒树状结构 (Inverted-tree Structure)**：DNS 域名空间被设计成一棵“倒长”的树，**根 (root)** 在最顶部。
* **层级限制 (128 Levels)**：这棵树极其深，理论上可以从 **Level 0 (根)** 一直延伸到 **Level 127**。
* **Level 0**：根域名，用一个点 `.` 表示。
* **Level 1**：顶级域名 (TLD)，如 `.cn`, `.edu`, `.com`。
* **Level 2**：二级域名，如 `tsinghua.edu.cn` 中的 `tsinghua`。



### 2. 标签与命名规则 (Label) (对应 image_291d6c.jpg)

这张图解释了域名中每一段（由点分隔的部分）的格式要求。

* **标签 (Label)**：树中的每个节点都有一个标签。
* **63 字符限制**：每个标签的最大长度为 **63 个字符**。
* *注：这是因为在 DNS 协议报文中，长度字段仅占 6 位，2 的 6 次方等于 64，去掉 0 后最大即为 63。*


* **根标签 (Null String)**：根节点的标签是一个空字符串。
* **唯一性要求**：在同一个父节点下的子节点必须拥有**不同的标签**，以保证全球路径唯一。

### 3. 分布式数据库 (DNS Partitioning) (对应 image_291abf.png)

这张图展示了 DNS 是如何解决“数据量太大”的问题的。

* **化整为零**：DNS 不会把全球所有的域名映射关系存放在一台电脑上。
* **分布式存储 (Distributed Database)**：它将海量信息切分成许多小块（称为 **Zone/区**），并分别存储在世界各地不同的 DNS 服务器中。
* **好处**：这样做不仅解决了单点故障问题，还大大提高了查询效率。

### 4. 为什么 Host 文件被淘汰了？ (对应 image_291aa3.png)

这张图对比了现代 DNS 与早期的 `host` 文件方案。

* **Host 文件的局限**：在互联网早期，所有映射都靠一个人工维护的 `host` 文件。
* **现代困境**：如今互联网规模巨大且瞬息万变，要求每台电脑都拥有一份包含全球所有地址的完整文件是**不可能的**。
* **Master 模式的终结**：图中展示了早期通过一台“Master”主机分发文件的模式，这在现代网络中完全无法扩展。

### 💡 核心参数速记表（常考）

| 参数 | 限制值 | 原因/备注 |
| --- | --- | --- |
| **单级标签长度** | **63** 字符 | 受协议报文 6 位长度字段限制 |
| **域名总长度** | **255** 字符 | 为了简化实现和内存分配 |
| **树的最大深度** | **128** 层 | 从 Level 0 到 Level 127 |
