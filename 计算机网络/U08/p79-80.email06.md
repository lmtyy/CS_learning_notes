这最后两张课件为你讲解了**邮件读取协议 (Mail Access Protocols)**，它们解决了邮件系统的“最后一公里”问题——即如何让收件人从服务器拿到属于自己的信件。

作为大学生，你需要重点掌握 **POP3** 和 **IMAP4** 的区别，这是计网期末考试的必考点。

---

### 1. 邮件读取协议：从“推”到“拉”

在之前的学习中，我们知道 **SMTP** 是一个“推 (Push)”协议，用于把邮件发出去。但邮件服务器通常是 7x24 小时运行的，而你的电脑（用户代理）并不是。

* **异步性**：邮件先由 SMTP 推送到接收方的邮件服务器，并存放在其**信箱 (Mailbox)** 中。
* **拉取 (Pull)**：当用户想要看信时，就需要使用 **POP3** 或 **IMAP4** 协议主动从服务器“拉取”下载邮件。

---

### 2. 核心协议对比：POP3 vs IMAP4

#### **POP3 (邮局协议版本 3)**

* **全称**：Post Office Protocol, version 3。
* **熟知端口**：基于 TCP，标准端口为 **110**。
* **工作特点**：
* **下载并删除**：默认情况下，用户把邮件下载到本地电脑后，服务器上的备份会被删除。
* **离线处理**：它假设用户在本地阅读邮件，不鼓励在服务器上管理邮件夹。



#### **IMAP4 (因特网邮件访问协议版本 4)**

* **全称**：Internet Mail Access Protocol, version 4。
* **工作特点**：
* **更强大、更复杂**：它允许用户在服务器上直接管理邮件。
* **多端同步**：你可以只下载邮件的首部，或者在服务器上创建文件夹。你在手机上标记为“已读”，电脑上也会同步显示“已读”。

### 3. 邮件系统完整流程图 (对应 image_52e4cb.jpg)

这张图总结了发送和接收邮件时协议的配合逻辑：

1. **发送阶段 (SMTP)**：发件人 User Agent -> 发件服务器 -> 接收服务器。
2. **存储阶段**：接收服务器将邮件放入 **Mail cache (邮件缓存/信箱)**。
3. **接收阶段 (POP3/IMAP4)**：收件人 User Agent 主动连接服务器的 **110 端口**，将邮件取回。
