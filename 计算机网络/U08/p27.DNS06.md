以下是详细解读：

### 1. 现实生活中的 DNS 配置 (右侧截图)

课件右上角的图片是你熟悉的 Windows **“Internet 协议 (TCP/IP) 属性”** 窗口。

* **手动设置 DNS**：当你勾选“使用下面的 DNS 服务器地址”时，你输入的 IP（如 `202.96.64.68`）就是你的**本地域名服务器 (Local Name Server)**。
* **作用**：每当你上网输入网址时，你的电脑第一步就是去问这台服务器：“请问这个域名的 IP 是多少？”。

---

### 2. 三种核心域名服务器 (重点考点)

课件列出了 DNS 体系结构中的三根支柱：

#### **A. 根域名服务器 (Root Name Server)**

* **互联网的顶点**：它是域名解析的起点。
* **职责**：它并不直接存储具体的网页 IP，而是**记录了所有顶级域名（如 .com, .cn, .edu）的 DNS 信息**。
* **数量**：全球共有 **13 台** 逻辑上的根服务器（编号 A 到 M）。

#### **B. 本地域名服务器 (Local Name Server)**

* **你的“专属管家”**：通常由你的 ISP（如电信、联通）或学校网络中心提供。
* **离你最近**：因为它距离用户较近，所以解析速度非常快。它是你电脑发起 DNS 查询时的第一个“求助对象”。

#### **C. 授权域名服务器 (Authoritative Name Server)**

* **权威终点**：这是真正存储特定主机域名与 IP 映射关系的地方。
* **注册登记**：**每一个主机名都必须在授权域名服务器处注册登记**。例如，清华大学的服务器 IP 就登记在清华大学自己的授权 DNS 服务器上。

---

### 💡 核心逻辑总结

当你在浏览器输入网址时，查询流程通常是：

1. **你的电脑** -> 问 **本地域名服务器**。
2. **本地域名服务器**（如果没有缓存） -> 问 **根域名服务器**。
3. **根域名服务器** -> 指向 **顶级域名服务器** -> 最终找到 **授权域名服务器**。
4. **授权域名服务器** -> 把最终的 **IP 地址** 告诉本地域名服务器，再传回给你。
