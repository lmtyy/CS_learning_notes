### 8.1 Client-Server Model (客户-服务器模型)

这是网络应用最主流的架构模式。

* **核心概念：**
* **Client (客户)：** 服务请求方（比如你的浏览器，或者你用 React 写的前端）。通常是主动发起通信的一方。
* **Server (服务器)：** 服务提供方（比如你用 Express 写的后端）。通常是被动的，一直运行，等待请求，并且有一个固定的 IP 地址。


* **重点理解：**
* C/S 模式 vs P2P (对等) 模式的区别。（虽然目录没写 P2P，但考试常考对比）。
* 服务器通常是长时间在线，且拥有固定 IP；客户端可以是间歇性连接，IP 可能是动态的。



### 8.2 Domain Name System (DNS, 域名系统)

互联网的“电话簿”。

* **核心功能：** 将人类可读的主机名（如 `google.com`）转换为机器可读的 IP 地址（如 `142.250.xxx.xxx`）。
* **常考点：**
* **分层结构：** 根域名服务器 -> 顶级域名服务器 (TLD) -> 权限域名服务器。
* **查询方式：** 递归查询 (Recursive) vs 迭代查询 (Iterative) 的区别。
* **缓存 (Caching)：** DNS 缓存在浏览器和本地的重要性，用于提高效率。
* **协议：** DNS 主要使用 **UDP** 协议（端口 53），但在某些特殊情况（如区域传送）下使用 TCP。



### 8.3 TELNET (远程终端协议)

早期的远程登录协议。

* **核心功能：** 允许用户通过网络登录到远程主机，像操作本地电脑一样操作远程电脑的命令行。
* **致命弱点（必考）：**
* **明文传输：** Telnet 发送的所有数据（包括用户名和密码）都是不加密的。
* **现状：** 现在几乎已经被 **SSH (Secure Shell)** 取代，因为 SSH 是加密的。但在教学中，Telnet 常作为理解远程登录原理的例子。



### 8.4 File Transfer Protocol (FTP, 文件传输协议)

用于在网络上进行文件传输。

* **核心机制（难点）：**
* FTP 使用 **两个并行的 TCP 连接**：
1. **控制连接 (Control Connection)：** 端口 21，用于发送指令（如登录、改变目录）。
2. **数据连接 (Data Connection)：** 端口 20，专门用于传文件数据。


* **带外传送 (Out-of-band)：** 因为控制指令和数据是分开传的，这叫带外传送。



### 8.5 Electronic Mail (电子邮件)

邮件系统比你想象的要复杂，涉及多个协议。

* **三大组件：** 用户代理 (User Agent)、邮件服务器 (Mail Server)、邮件协议。
* **核心协议（易混淆）：**
* **SMTP (Simple Mail Transfer Protocol)：** 用于**发送**邮件（推）。从你的电脑推到你的邮件服务器，再推到对方的邮件服务器。
* **POP3 / IMAP：** 用于**接收**邮件（拉）。从对方的邮件服务器拉取到你的电脑上。
* *POP3：* 下载后通常会删除服务器副本（比较老旧）。
* *IMAP：* 在服务器上管理邮件，多设备同步（现在主流）。





### 8.6 WWW (万维网)

这块和你现在学的 Web 开发最相关。

* **核心协议：** **HTTP (HyperText Transfer Protocol)**。
* **常考重点：**
* **URL：** 统一资源定位符。
* **HTTP 是无状态的 (Stateless)：** 服务器不记得你是谁。为了解决这个问题，引入了 **Cookie** 和 **Session**（这也是 Web 开发面试的必考题）。
* **持久连接 vs 非持久连接：** HTTP/1.0 默认非持久（传完一个对象就断开 TCP），HTTP/1.1 默认持久（Keep-Alive，传完不马上断，复用连接）。
