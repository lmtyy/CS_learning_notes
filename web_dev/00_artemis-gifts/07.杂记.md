### 📝 笔记一：部署配置避坑指南 (DevOps)

**核心口诀：** “构建归构建，发布归系统，目录要填对。”

1. **Output Directory 是生命线：**
* Vite 项目打包后一定会生成一个 `dist` 文件夹。
* 在 Cloudflare/Vercel 的设置里，**Build output directory** 必须填 **`dist`**。
* *错误后果：* 填空或填错，服务器就会直接把源代码扔上去，导致你看到 "Hello World" 或者空白页。


2. **不要画蛇添足写 Deploy 命令：**
* Cloudflare Pages 是自动化托管，它会自动检测 `dist` 目录并上传。
* **千万不要**在 Deploy command 里填 `wrangler deploy`，那是给后端代码用的。留空 (`Optional`) 才是标准答案。


3. **SPA 的 404 魔法：**
* React 单页应用必须在 `public` 文件夹放一个 **`_redirects`** 文件。
* 内容写 `/* /index.html 200`，防止刷新页面时变成 404。

### 📝 笔记二：优雅的前端架构 (Architecture)

**核心口诀：** “数据要分离，资源用 Import。”

1. **文案与代码分离 (Config Driven)：**
* 不要把情书、祝福语硬编码在 JSX 里。
* 建立 **`src/content.js`** 存放所有文字。这样改错别字不用动逻辑代码，结构也更清晰，甚至以后可以轻松做成多语言版。


2. **图片资源管理：**
* **推荐：** 把图片放在 `src/assets`，在组件里用 `import img from ...` 引入。
* *好处：* 打包工具（Vite）会自动给文件名加哈希值（如 `bg-x8d2.png`），完美解决“换了图但浏览器还有缓存”的问题。
* *避坑：* 线上服务器（Linux）对**文件名大小写极度敏感**，本地 `Image/bg.png` 能跑，线上必须一模一样，写成 `images/BG.png` 直接报错。

---

### 📝 笔记三：CSS 布局微调 (Layout & UI)

**核心口诀：** “盒子要算准，图片要去边。”

1. **消灭幽灵白边：**
* 图片 `<img>` 默认底部会有 3-5px 的空隙（为了对齐字母基线）。
* *解法：* 给图片加上 `display: block;` 或者 `vertical-align: bottom;`。


2. **Flexbox 里的“自动填充”：**
* 当你想让内容区占满“标题栏剩下的所有空间”时，不要用 `height: calc(100% - 50px)`。
* *解法：* 父容器设为 `display: flex; flex-direction: column;`，内容区设为 **`flex: 1;`**。这才是响应式布局的正解。


3. **手机端适配 (Media Query)：**
* PC 端项目在手机上会错位。
* *兜底大招：* 用 `@media (max-width: 768px)` 加一个全屏遮罩层，提示用户“请使用电脑访问”，保证第一印象不扣分。
