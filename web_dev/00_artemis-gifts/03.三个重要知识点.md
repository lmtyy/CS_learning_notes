### 1. 父子组件的“红线电话” (Props & Callbacks)

你刚才在 `Desktop.jsx` 里写了 `onClose={() => setShowChristmas(false)}`，然后在 `Christmas2025.jsx` 里用了 `({ onClose })`。

* **知识点：** **数据向下流，命令向上走。**
* **向下：** 父组件 (Desktop) 把数据或函数传给子组件，这叫 **Props**。
* **向上：** 子组件 (Window) 不能直接修改父组件的状态（它不能直接改 `showChristmas`）。它必须**“打电话”**（调用父组件传下来的 `onClose` 函数）请求父组件自己修改。


* **开发直觉：** 如果你发现子组件里缺东西，或者没法关掉窗口，检查一下**“电话线”**（Props）有没有接好。

### 2. JSX 不是 HTML (CamelCase & ClassName)

虽然代码看起来像 HTML，但它是 JavaScript。

* **陷阱：**
* HTML 里写 `class="box"`，JSX 里必须写 **`className="box"`**。
* HTML 里写 `onclick="..."`，JSX 里必须写 **`onClick={...}`** (驼峰命名)。
* HTML 里标签可以不闭合 `<br>`，JSX 里**必须闭合** `<br />` 或 `<img />`。


* **开发直觉：** 如果报错说 `Expected corresponding JSX closing tag`，通常就是标签没闭合。

### 3. Vite 的资源引用魔法

你刚才处理 MP3 时已经接触过了。

* **知识点：** 在 React/Vite 里，图片和音乐不能只写一个字符串路径 `"../assets/tree.png"`。
* **正确做法：** 必须用 **`import`** 把它当成一个模块导入。
```javascript
import treeImg from '../assets/images/tree.png';
// 然后使用
<img src={treeImg} />

```


* **开发直觉：** 如果图片裂了或者不显示，先看有没有 `import`。
