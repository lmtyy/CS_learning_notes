### 📝 技术笔记 (二)：域名、DNS 与 SSL 进阶配置

**核心逻辑：**
域名（Domain）是门牌号，DNS 是指路牌，SSL 是安全锁。我们需要把买来的域名“过继”给 Cloudflare 管理，这样 Cloudflare 既能帮你指路（加速），又能帮你加密（安全）。

#### 1. 域名注册策略 (Domain Strategy)

* **为什么首选国外注册商 (NameSilo/Namecheap)？**
* **核心原因：** 避开繁琐的备案（ICP Filing）。国内注册的域名如果解析到国外服务器，经常会被要求备案或暂停解析。
* **隐私保护：** 国外大厂通常免费赠送 WHOIS Privacy，保护你的真实姓名和住址不被公开查询。


* **后缀选择：**
* `.com` 是永远的神，但 `.top`, `.xyz`, `.love` 等非主流后缀在个人项目上性价比极高（首年只需 $1-$2），且没有任何功能上的区别。



#### 2. DNS 托管权移交 (The NS Handover)

这是新手最容易晕的地方。你买域名的地方（NameSilo）和管理域名的地方（Cloudflare）可以是分离的。

* **Nameservers (NS 记录)：** 域名的“最高指挥权”。
* **操作流程：**
1. 在 Cloudflare 添加站点 (Add Site)，获取两个 `ns.cloudflare.com` 地址。
2. 去 NameSilo 把默认的 NS 记录**全删掉**，填入 Cloudflare 给你的那两个。


* **目的：** 让 Cloudflare 接管 DNS 解析。这样你可以享受它的全球 CDN 加速和免费的 SSL 证书。

#### 3. 域名绑定的“双层逻辑”

Cloudflare 的架构分两层，弄混了就会报错：

* **第一层：进大门 (Add Site)**
* 必须先在 Cloudflare **首页** 添加域名 (`arte-desk.top`)。如果不加，直接去 Pages 绑定会报 *"Not a registered domain"*。


* **第二层：进房间 (Pages Custom Domain)**
* 在 **Pages 项目** 里绑定域名。
* **常见报错：** *"This domain is already in use"*。
* **原因：** DNS 记录里有旧的占位符（通常是 NameSilo 默认生成的 A 记录或 CNAME）。
* **解法：** 去 DNS 设置里把所有关于这个域名的旧记录**统统删掉**，再重新绑定。



#### 4. HTTPS 与 SSL 避坑 (Security)

* **经典报错：Error 525 (SSL Handshake Failed)**
* **含义：** 浏览器找到了 Cloudflare，但 Cloudflare 和你的服务器（Pages）之间握手失败，或者证书还没发下来。


* **核心解法：**
1. **让子弹飞一会儿：** 新域名的证书颁发和全球生效需要时间（15分钟 - 1小时）。这是最常见的原因。
2. **加密模式 (Encryption Mode)：**
* **Flexible (灵活):** 浏览器到 CF 是加密的，CF 到服务器是明文的。（不推荐，容易导致重定向循环）。
* **Full (完全):** 全程加密。**这是 Cloudflare Pages 的推荐设置**。
* **Full (Strict):** 需要源服务器有受信任的证书。（Pages 自动管理，通常选 Full 即可）。
