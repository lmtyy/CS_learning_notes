### 第一部分：工程化与环境治理 (Engineering & Environment)

这部分内容虽然不直观地体现在界面上，但它是你能顺利写代码的前提。你解决了一个非常经典的“依赖地狱”问题。

#### 1. 版本兼容性与依赖管理 (Dependency Management)

* **遇到的问题：** `npm run dev` 后页面全白，控制台报错 `findDOMNode is not a function`。
* **原因分析：**
* **React 19 (最新版)**：为了性能和架构，删除了旧 API (`findDOMNode`)。
* **React95 (第三方库)**：底层代码较老，依赖这个旧 API。
* **结论：** “最新”不一定等于“最好/最合适”。软件工程中经常需要为了兼容生态而进行**降级 (Downgrade)**。


* **解决方案：**
* 强制指定版本安装：`npm install react@18.3.1 styled-components@5.3.11`。
* **清理缓存：** 遇到诡异问题先删缓存 `rm -rf node_modules/.vite`。



#### 2. React 严格模式 (Strict Mode)

* **知识点：** `<StrictMode>` 是 React 的开发工具，它会故意运行组件两次来查找潜在 Bug。
* **实战应用：**
* 在 `main.jsx` 中，我们删除了包裹在 `<App />` 外面的 `<StrictMode>`。
* **理由：** 对于使用老旧 UI 库（如 `react95`）的项目，严格模式的检查过于激进，会导致库内部报错。在快速开发原型时，关闭它是合理的工程决策。



#### 3. 资源引入规则 (Asset Imports in Vite)

* **知识点：** 在现代前端构建工具（如 Vite/Webpack）中，**图片、音频、字体都是模块**。
* **错误写法：** `<img src="../assets/tree.png" />` (这只是个字符串，浏览器找不到文件)。
* **正确写法：**
```javascript
import treeImg from '../assets/images/tree.png'; // 先引入
<img src={treeImg} /> // 后使用变量

```


* **原理：** `import` 会告诉 Vite 把这个文件打包、压缩、加哈希值（防缓存），并返回最终的 URL 给 JS。

#### 4. 路径引用 (Path Resolution)

* **遇到的问题：** `import ChristmasCollage` 时报错找不到文件。
* **排查口诀：**
* **./** 代表同级目录。
* **../** 代表向上一级。
* **../../** 代表向上两级。


* **技巧：** 如果搞不清层级，利用 VS Code 的“快速修复 (Quick Fix)”功能自动导入，避免手写路径错误。
