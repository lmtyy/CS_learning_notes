## 操作顺序必须是：

1. **先有本地仓库**
   ```bash
   git init  # 或者在本地clone一个仓库
   ```
   
2. **然后才能添加远程仓库**
   ```bash
   git remote add origin [url]
   ```

## 为什么需要这个顺序？

### 🏗️ 建筑比喻：
- 你得先有自己的房子（本地仓库）
- 然后才能设置通往其他地方的通道（远程仓库）

### 💻 实际操作验证：
如果你在没有本地仓库的目录下运行：
```bash
git remote add skeleton https://github.com/xxx/xxx.git
```
会得到错误：`not a git repository`

## 两种常见的使用场景：

### 场景1：从零开始新项目
```bash
# 1. 创建项目目录
mkdir my-project
cd my-project

# 2. 初始化本地仓库
git init

# 3. 添加远程仓库
git remote add origin https://github.com/yourname/my-project.git
```

### 场景2：参与已有项目
```bash
# 1. 直接克隆（这会自动创建本地仓库+添加远程）
git clone https://github.com/someproject/repo.git
cd repo

# 此时已经自动配置了远程仓库
git remote -v  # 可以看到origin
```
