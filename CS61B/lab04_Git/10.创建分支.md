创建分支在 Git 中有几种方法，让我为你详细讲解：

## 🌱 创建分支的基本命令

### 方法1：创建并切换到新分支
```bash
git switch -c [新分支名]
```
或者（旧版本 Git）：
```bash
git checkout -b [新分支名]
```

### 方法2：先创建分支，再切换
```bash
# 创建分支
git branch [新分支名]

# 切换到该分支
git switch [新分支名]
```

## 🎯 实际例子

### 例子1：从当前位置创建分支
```bash
# 假设当前在 master 分支，想创建开发新功能的分支
git switch -c feature/login-system
```

### 例子2：从特定提交创建分支
```bash
# 从某个提交创建分支
git branch bugfix-branch e137e9b
```

## 📊 基于图示的具体操作

从你提供的图片来看：

```bash
# 当前在 cool_branch，想基于 master 创建新分支
git switch master              # 先切换到 master
git switch -c new_feature      # 从 master 创建新分支

# 或者直接从当前分支创建
git switch -c another_branch   # 从当前分支创建
```

## 🔍 查看分支信息

```bash
# 查看所有分支（当前分支前有 * 号）
git branch

# 查看分支及最后提交信息
git branch -v

# 查看所有分支（包括远程）
git branch -a
```

## 🛠️ 分支命名建议

- **功能分支**：`feature/用户登录`
- **修复分支**：`bugfix/修复支付问题`  
- **发布分支**：`release/v2.0`
- **热修复**：`hotfix/紧急安全修复`

## 📝 完整工作流程示例

```bash
# 1. 在 master 分支上
git switch master

# 2. 创建新功能分支
git switch -c feature/cool-new-feature

# 3. 开发并提交更改
git add .
git commit -m "实现酷炫新功能"

# 4. 完成后切换回 master
git switch master
```

## 💡 重要提示

- 创建分支时是基于**当前所在位置**的提交
- 新分支包含当前分支的所有提交历史
- 分支之间完全独立，互不干扰
- 可以随时在不同分支间切换

这样你就可以轻松管理多个开发任务了！🎉
